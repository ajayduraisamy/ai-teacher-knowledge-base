## Definition

Matplotlib is the foundational data visualization library for Python. It provides a MATLAB-like interface for creating static, animated, and interactive plots. The library is built around a hierarchy of objects: a `Figure` sits at the top, containing one or more `Axes` objects, each of which is the actual plotting area with tick marks, labels, and plotted data.

The two primary APIs are:

1. **pyplot (state-based):** A collection of module-level functions (`plt.plot()`, `plt.xlabel()`) that implicitly create and track a "current" figure and axes. Best for quick, exploratory plotting.
2. **Object-Oriented (OOP):** Explicitly create `Figure` and `Axes` objects, then call methods on them (`ax.plot()`, `ax.set_xlabel()`). Essential for complex multi-plot layouts, embedding, and production code.

The conceptual hierarchy is: `Figure` → `Axes` → `Axis` → ticks/labels, with `Artist` objects (lines, patches, text) rendered onto the `Axes`.

## Intuition

Think of Matplotlib like a painter's canvas:
- The **Figure** is the entire sheet of paper (or screen). It holds everything.
- The **Axes** is the region where the actual data drawing happens — the rectangular area with x and y axes.
- **Axis** objects are the rulers along the edges of the Axes, marking tick positions and labels.
- **Artists** are everything drawn on the Axes: lines, bars, markers, text annotations, etc.

When you call `plt.plot()`, the pyplot module finds (or creates) the current Axes and adds a line artist to it. In OOP style, you explicitly get a reference to the Axes and call `ax.plot()` — giving you fine-grained control, especially when managing multiple subplots.

## Why This Concept Matters

Matplotlib is the bedrock of Python visualization. Nearly every other visualization library (Seaborn, Pandas `.plot()`, etc.) is built on top of Matplotlib. Understanding its OOP architecture allows you to:
- Create publication-quality figures with precise control over every visual element
- Build complex multi-panel layouts
- Debug and customize plots generated by higher-level libraries
- Transition seamlessly between exploratory analysis and production visualization

Without Matplotlib fundamentals, you're limited to whatever defaults a higher-level library provides. With them, you can produce any static plot imaginable — a skill essential for data scientists, ML engineers, and researchers.

## Code Examples

### Example 1: Simple pyplot style
```python
import matplotlib.pyplot as plt
import numpy as np

x = np.linspace(0, 10, 100)
y = np.sin(x)

plt.figure(figsize=(8, 4))
plt.plot(x, y, label='sin(x)')
plt.title('Simple Sine Wave')
plt.xlabel('x')
plt.ylabel('sin(x)')
plt.legend()
plt.grid(True)
plt.show()
```
```
# Output: A window/png showing a sine wave from 0 to 10 with grid and legend.
```

### Example 2: Object-Oriented (OOP) style
```python
fig, ax = plt.subplots(figsize=(8, 4))
x = np.linspace(0, 10, 100)
ax.plot(x, np.sin(x), label='sin(x)')
ax.set_title('OOP Style Sine Wave')
ax.set_xlabel('x')
ax.set_ylabel('sin(x)')
ax.legend()
ax.grid(True)
plt.show()
```
```
# Output: Identical visual to Example 1, but using explicit Axes methods.
```

### Example 3: Multiple subplots with OOP
```python
fig, axes = plt.subplots(1, 2, figsize=(10, 4))
x = np.linspace(0, 10, 100)

axes[0].plot(x, np.sin(x), 'b-')
axes[0].set_title('Sine')

axes[1].plot(x, np.cos(x), 'r--')
axes[1].set_title('Cosine')

fig.suptitle('Trigonometric Functions')
plt.tight_layout()
plt.show()
```
```
# Output: Two side-by-side plots showing sine (blue solid) and cosine (red dashed).
```

### Example 4: Saving figures to disk
```python
fig, ax = plt.subplots()
x = np.linspace(0, 5, 50)
ax.plot(x, x**2, label='x²')
ax.plot(x, x**3, label='x³')
ax.set_title('Polynomials')
ax.legend()

fig.savefig('polynomials.png', dpi=300, bbox_inches='tight')
fig.savefig('polynomials.pdf', format='pdf')
print("Figures saved.")
```
```
# Output: Figures saved.
# Two files created: polynomials.png (300 DPI) and polynomials.pdf (vector).
```

### Example 6: Customizing figure and Axes properties in depth
```python
fig = plt.figure(figsize=(10, 6))
ax = fig.add_subplot(111)

x = np.arange(0, 10, 0.1)
y1 = np.sin(x)
y2 = np.cos(x)

ax.plot(x, y1, color='#E74C3C', linewidth=2, alpha=0.8, label='sin')
ax.plot(x, y2, color='#3498DB', linewidth=2, alpha=0.8, label='cos')

ax.set_xlim(0, 10)
ax.set_ylim(-1.5, 1.5)
ax.set_xticks(np.arange(0, 11, 2))
ax.set_yticks([-1, 0, 1])
ax.set_xlabel('Time (s)', fontsize=12)
ax.set_ylabel('Amplitude', fontsize=12)
ax.set_title('Trigonometric Functions with Custom Axes', fontsize=14, fontweight='bold')
ax.legend(loc='upper right', frameon=True, shadow=True)
ax.grid(True, linestyle=':', alpha=0.7)
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

plt.tight_layout()
plt.show()
```
```
# Output: A styled sine/cosine plot with limited axes, custom ticks, thin grid, and hidden top/right spines.
```

## Common Mistakes

1. **Mixing pyplot and OOP styles inconsistently.** Avoid calling `plt.xlabel()` after creating `fig, ax = plt.subplots()`. Use `ax.set_xlabel()` consistently in OOP mode.
2. **Calling `plt.show()` before `plt.savefig()`.** On some backends, `plt.show()` clears the figure, resulting in an empty saved file. Always save before showing, or specify `bbox_inches='tight'`.
3. **Forgetting `fig.tight_layout()` or `plt.tight_layout()`.** Subplot labels, titles, and tick labels frequently overlap; tight_layout adjusts spacing automatically.
4. **Creating multiple figures without tracking them.** Each call to `plt.figure()` creates a new figure. Unreferenced figures accumulate in memory. Use the OOP approach and store references.
5. **Assuming the "current axes" after closing a figure.** If an interactive window is closed, the current axes reference may become invalid. Always use explicit `ax` objects.
6. **Setting `figsize` in inches without considering DPI.** `figsize=(8,6)` at 100 DPI gives 800x600 pixels. At 300 DPI it gives 2400x1800 pixels. Both are the same logical size on screen but different file resolutions.
7. **Using `plt.subplot()` (single, adds one at a time) instead of `plt.subplots()` (bulk create all).** `plt.subplot(2,2,1)` followed by three more calls is error-prone; prefer `fig, axes = plt.subplots(2,2)` for cleaner code.

## Interview Questions

### Beginner - 5

1. **Q:** What is the difference between `plt.plot()` and `ax.plot()`?  
   **A:** `plt.plot()` is the pyplot state-machine interface that operates on the "current" axes (creating one if needed). `ax.plot()` is the object-oriented method called on a specific Axes instance, giving explicit control over which subplot receives the plot.

2. **Q:** How do you save a Matplotlib figure to a file?  
   **A:** Call `fig.savefig('filename.png', dpi=300, bbox_inches='tight')` before `plt.show()`. Supported formats include PNG, PDF, SVG, and EPS.

3. **Q:** What does `plt.subplots(2, 3)` return?  
   **A:** It returns a tuple `(Figure, ndarray of Axes)` where the Figure contains a 2-by-3 grid of Axes objects, accessible as `axes[0,0]`, `axes[0,1]`, etc.

4. **Q:** How do you add a title and axis labels in OOP style?  
   **A:** Using `ax.set_title('Title')`, `ax.set_xlabel('X-axis')`, `ax.set_ylabel('Y-axis')`, or a single call to `ax.set(title='...', xlabel='...', ylabel='...')`.

5. **Q:** What is the `Figure` vs `Axes` distinction?  
   **A:** `Figure` is the top-level container (the entire window/canvas). `Axes` is the plotting region within the figure with its own coordinate system (x/y axis). One figure can contain many axes.

### Intermediate - 5

1. **Q:** How can you share x or y axes across subplots?  
   **A:** Pass `sharex=True` or `sharey=True` / `'row'` / `'col'` to `plt.subplots()`:  
   `fig, axes = plt.subplots(2, 2, sharex=True, sharey='row')`. This synchronizes axis limits and scales.

2. **Q:** What is the purpose of `plt.tight_layout()` and when would you use `constrained_layout` instead?  
   **A:** `tight_layout()` automatically adjusts subplot parameters to prevent overlapping elements. `constrained_layout` is a newer, more flexible system that adjusts during draw time (set via `fig, ax = plt.subplots(constrained_layout=True)`), better for interactive use and complex grids.

3. **Q:** How do you add a colorbar to a plot?  
   **A:** For a scalar-mappable object (e.g., from `imshow`, `contourf`, or `scatter`), use `fig.colorbar(scalar_mappable, ax=ax)`. For a single axes figure: `plt.colorbar()`.

4. **Q:** Explain the `bbox_inches='tight'` parameter in `savefig()`.  
   **A:** It removes extra whitespace padding around the figure by calculating the tight bounding box of all artists. Essential for saving figures that will be placed in documents or on web pages.

5. **Q:** How does Matplotlib handle figure DPI and what is the `figsize` unit?  
   **A:** DPI (dots per inch) controls resolution. `figsize=(w, h)` is in inches. Pixel dimensions = figsize * DPI. Default DPI is 100 for screen; 300+ is typical for publication.

### Advanced - 3

1. **Q:** How can you embed Matplotlib in a custom GUI application (e.g., PyQt or Tkinter)?  
   **A:** Use `matplotlib.backends.backend_qt5agg.FigureCanvasQTAgg(fig)` or `backend_tkagg.FigureCanvasTkAgg(fig)` to embed the figure in the GUI. The figure must be drawn with `canvas.draw()`.

2. **Q:** Explain the Matplotlib Artist hierarchy and how `setp()` works.  
   **A:** Every element (Figure, Axes, Line2D, Text, etc.) is an `Artist`. The `plt.setp()` function introspects an artist and can set multiple properties at once. Artists are organized in a tree: Figure contains Axes, Axes contains Axis spines, ticks, and plotted artists.

3. **Q:** How would you create a custom projection (e.g., polar or 3D) and add custom transforms?  
   **A:** Use `fig.add_subplot(projection='polar')` or `projection='3d'` (requires `from mpl_toolkits.mplot3d import Axes3D`). Custom transforms subclass `matplotlib.transforms.Transform` and override `transform_non_affine` and `inverted`.

## Summary

Matplotlib is the foundational Python visualization library. Its two APIs — pyplot (quick, state-based) and OOP (explicit, controllable) — serve different needs. The core hierarchy (Figure → Axes → Axis → Artists) gives you complete control over every aspect of a plot. Mastering OOP Matplotlib unlocks the ability to create complex, publication-quality multi-panel figures and is a prerequisite for advanced visualization work in Python.

## Key Takeaways

- Use OOP style (`fig, ax = plt.subplots()`) for anything beyond the simplest one-off plot
- Always call `plt.show()` after `plt.savefig()`, or risk empty files
- `plt.tight_layout()` prevents overlapping elements in multi-subplot figures
- Know the hierarchy: `Figure` contains `Axes`, which contains plotted `Artists`
- Matplotlib is the engine under Seaborn, Pandas plotting, and many other tools
