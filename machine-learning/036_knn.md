## Definition

k-Nearest Neighbors (kNN) is a non-parametric, instance-based learning algorithm used for classification and regression. For a query point, kNN identifies the k closest labeled training examples in feature space and predicts based on their labels — majority vote for classification, average for regression. Being a lazy learner, kNN stores the entire training set and performs all computation at inference time.

Given a training set $\mathcal{D} = \{(x_i, y_i)\}_{i=1}^n$ and a query $x_q$, kNN finds the set $\mathcal{N}_k(x_q)$ of k indices with smallest distance $d(x_q, x_i)$. For classification:

$$\hat{y}_q = \arg\max_c \sum_{i \in \mathcal{N}_k(x_q)} \mathbb{I}(y_i = c)$$

For regression:

$$\hat{y}_q = \frac{1}{k} \sum_{i \in \mathcal{N}_k(x_q)} y_i$$

## Intuition

kNN embodies the principle of "birds of a feather flock together." Points close in feature space are assumed to share the same label or similar target values. Think of it as consulting your k nearest neighbors in a new city to decide which restaurant to visit: you ask the people whose tastes most resemble yours.

The parameter k controls flexibility. With k=1, the model draws tight, highly specific boundaries around every training point — low bias but high variance, prone to overfitting. With large k, predictions smooth over many neighbors, reducing variance but potentially underfitting by washing out local patterns. The optimal k balances these forces at the bias-variance sweet spot.

## Why This Concept Matters

kNN is one of the simplest and most interpretable ML algorithms, serving as a baseline that often surprises practitioners with strong performance on low-dimensional problems. It is widely used in recommendation systems, anomaly detection, missing value imputation, and as a building block in more complex algorithms. Understanding kNN's weaknesses — computational cost at inference, sensitivity to scale and irrelevant features — teaches fundamental lessons about data preprocessing, the curse of dimensionality, and the tradeoffs that apply across all of machine learning.

## Mathematical Explanation

### Distance-Weighted Voting

Basic kNN gives each neighbor equal vote. Distance weighting improves this:

$$\hat{y}_q = \arg\max_c \sum_{i \in \mathcal{N}_k(x_q)} w_i \cdot \mathbb{I}(y_i = c)$$

where $w_i = 1 / d(x_q, x_i)$ or $w_i = 1 / (d(x_q, x_i)^2)$. Closer neighbors exert more influence.

### Bias-Variance Decomposition

For regression with kNN using squared error loss:

- **Bias**: $\left(\mathbb{E}[\hat{f}(x)] - f(x)\right)^2$ — increases with k as the estimate smooths over more distant points
- **Variance**: $\mathbb{E}\left[(\hat{f}(x) - \mathbb{E}[\hat{f}(x)])^2\right]$ — decreases with k since more neighbors average out noise

Expected test error at a point x:

$$\text{MSE}(x) = \text{Bias}^2(x) + \text{Var}(x) + \sigma_\epsilon^2$$

### Curse of Dimensionality

In high dimensions, the volume of space grows exponentially, making all points appear equally distant. For d-dimensional data, the expected distance ratio between farthest and nearest points approaches 1 as d grows:

$$\lim_{d \to \infty} \frac{d_{\max} - d_{\min}}{d_{\min}} = 0$$

This means kNN's notion of "neighborhood" breaks down in high dimensions, requiring exponentially more training data to maintain density.

### Effective Parameters

Though non-parametric, kNN's model complexity is controlled by k. The effective number of parameters is approximately $n/k$, since the training set is partitioned into roughly $n/k$ neighborhoods.

## Code Examples

### Example 1: Basic kNN Classification with Decision Boundary

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.neighbors import KNeighborsClassifier
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# Create synthetic dataset
X, y = make_classification(n_samples=300, n_features=2, n_classes=3,
                           n_redundant=0, n_clusters_per_class=1,
                           random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Scale features
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

# Train kNN
knn = KNeighborsClassifier(n_neighbors=5, weights='uniform')
knn.fit(X_train_scaled, y_train)
accuracy = knn.score(X_test_scaled, y_test)

print(f"Accuracy with k=5: {accuracy:.3f}")
# Output: Accuracy with k=5: 0.911

# Decision boundary visualization
x_min, x_max = X_train_scaled[:, 0].min() - 1, X_train_scaled[:, 0].max() + 1
y_min, y_max = X_train_scaled[:, 1].min() - 1, X_train_scaled[:, 1].max() + 1
xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.02),
                     np.arange(y_min, y_max, 0.02))
Z = knn.predict(np.c_[xx.ravel(), yy.ravel()])
Z = Z.reshape(xx.shape)

plt.figure(figsize=(8, 6))
plt.contourf(xx, yy, Z, alpha=0.3, cmap='viridis')
plt.scatter(X_train_scaled[:, 0], X_train_scaled[:, 1], c=y_train,
            edgecolors='k', cmap='viridis', alpha=0.8)
plt.title('kNN Decision Boundary (k=5, uniform weights)')
plt.xlabel('Feature 1 (scaled)')
plt.ylabel('Feature 2 (scaled)')
plt.tight_layout()
plt.savefig('knn_boundary.png', dpi=150)
print("Decision boundary plot saved.")
# Output: Decision boundary plot saved.
```

### Example 2: Effect of k on Bias-Variance Tradeoff

```python
import numpy as np
from sklearn.neighbors import KNeighborsRegressor
from sklearn.model_selection import cross_val_score
from sklearn.datasets import make_regression

X, y = make_regression(n_samples=500, n_features=10, noise=15, random_state=42)

k_values = [1, 2, 3, 5, 10, 15, 20, 30, 50, 75, 100, 150, 200]
train_scores = []
val_scores = []

for k in k_values:
    knn = KNeighborsRegressor(n_neighbors=k)
    train_score = knn.fit(X, y).score(X, y)
    val_score = cross_val_score(knn, X, y, cv=5, scoring='r2').mean()
    train_scores.append(train_score)
    val_scores.append(val_score)
    print(f"k={k:3d} | Train R²={train_score:.4f} | Val R²={val_score:.4f}")

# Output:
# k=  1 | Train R²=1.0000 | Val R²=0.4312
# k=  2 | Train R²=0.9024 | Val R²=0.5901
# k=  3 | Train R²=0.8329 | Val R²=0.6498
# k=  5 | Train R²=0.7234 | Val R²=0.6942
# k= 10 | Train R²=0.5881 | Val R²=0.7231
# k= 15 | Train R²=0.5109 | Val R²=0.7300
# k= 20 | Train R²=0.4593 | Val R²=0.7335
# k= 30 | Train R²=0.3933 | Val R²=0.7294
# k= 50 | Train R²=0.3137 | Val R²=0.7184
# k= 75 | Train R²=0.2567 | Val R²=0.6982
# k=100 | Train R²=0.2144 | Val R²=0.6801
# k=150 | Train R²=0.1615 | Val R²=0.6380
# k=200 | Train R²=0.1254 | Val R²=0.5929

# Find optimal k
best_idx = np.argmax(val_scores)
print(f"Optimal k={k_values[best_idx]} with R²={val_scores[best_idx]:.4f}")
# Output: Optimal k=20 with R²=0.7335
```

### Example 3: Distance Weighting vs Uniform

```python
import numpy as np
from sklearn.neighbors import KNeighborsClassifier
from sklearn.datasets import make_moons
from sklearn.model_selection import cross_val_score

X, y = make_moons(n_samples=400, noise=0.15, random_state=42)

knn_uniform = KNeighborsClassifier(n_neighbors=15, weights='uniform')
knn_distance = KNeighborsClassifier(n_neighbors=15, weights='distance')

scores_uniform = cross_val_score(knn_uniform, X, y, cv=5)
scores_distance = cross_val_score(knn_distance, X, y, cv=5)

print(f"Uniform weights:  Mean={scores_uniform.mean():.3f}  Std={scores_uniform.std():.3f}")
print(f"Distance weights: Mean={scores_distance.mean():.3f}  Std={scores_distance.std():.3f}")
# Output:
# Uniform weights:  Mean=0.967  Std=0.014
# Distance weights: Mean=0.972  Std=0.012

# With noisy outlier added
np.random.seed(42)
X_noise = np.vstack([X, [[2.5, 2.5]]])
y_noise = np.append(y, 0)

knn_u = KNeighborsClassifier(n_neighbors=15, weights='uniform').fit(X_noise, y_noise)
knn_d = KNeighborsClassifier(n_neighbors=15, weights='distance').fit(X_noise, y_noise)

# Predict at the outlier's location
pred_u = knn_u.predict([[2.5, 2.5]])
pred_d = knn_d.predict([[2.5, 2.5]])
print(f"Uniform prediction at outlier: {pred_u[0]}")
print(f"Distance-weighted prediction at outlier: {pred_d[0]}")
# Output:
# Uniform prediction at outlier: 1
# Distance-weighted prediction at outlier: 0
```

import numpy as np

## Common Mistakes

1. **Forgetting to scale features.** kNN relies on distances; a feature with large numerical range (e.g., salary in dollars) will dominate features with small ranges (e.g., age in years). Always standardize or normalize.

2. **Choosing k without cross-validation.** Picking k=1 because "it's simplest" leads to overfitting. Picking k=n underfits. Use cross-validation to tune.

3. **Using kNN on high-dimensional data without dimensionality reduction.** The curse of dimensionality causes distances to become meaningless beyond ~20 features. Apply PCA or feature selection first.

4. **Using uniform weights when data is noisy.** Distance-weighted voting makes kNN more robust to outliers by reducing the influence of distant, potentially noisy neighbors.

5. **Ignoring class imbalance.** With imbalanced classes, majority classes dominate kNN predictions. Use stratified sampling, class weights, or resampling.

6. **Applying kNN to very large datasets without approximation.** kNN stores all training data and compares against every point at inference, scaling as O(nd). Use KDTree, BallTree, or approximate nearest neighbor (ANN) algorithms for large-scale deployment.

7. **Assuming k must be odd.** While odd k breaks ties in binary classification, tie-breaking rules in sklearn handle even k automatically. For multiclass, parity is irrelevant.

## Interview Questions

### Beginner

1. What does "lazy learning" mean, and why is kNN considered a lazy learner?

kNN is a lazy learner because it does not build an explicit model during training; it simply stores the training data and defers all computation until inference time. This contrasts with eager learners like decision trees or neural networks that train a model upfront.

2. How do you choose the optimal value of k in kNN?

Use cross-validation to evaluate performance across a range of k values. Plot validation error vs. k — choose k at the elbow where error stabilizes. Alternatively, use the square root rule of thumb: $k \approx \sqrt{n}$.

3. Why is feature scaling important for kNN?

kNN computes distances between points. Features with larger magnitudes dominate the distance calculation, making the algorithm effectively ignore smaller-scale features. Scaling ensures all features contribute equally.

4. What is the time complexity of kNN at training and inference?

Training: O(1) (lazy). Inference: O(nd + nk) for brute-force — n comparisons, each costing O(d), plus O(nk) for voting/scoring.

5. Can kNN be used for regression?

Yes. kNN regression predicts the average (or weighted average) of the target values of the k nearest neighbors. sklearn provides KNeighborsRegressor.

### Intermediate

1. Explain the bias-variance tradeoff in kNN as a function of k.

Small k: low bias (model fits training data tightly), high variance (sensitive to noise in individual points). Large k: high bias (overly smooth, misses local patterns), low variance (stable across different training sets). The optimal k minimizes total expected error.

2. How does the curse of dimensionality affect kNN? What strategies mitigate it?

In high dimensions, distances between all points converge, making neighborhoods meaningless. The volume grows exponentially, requiring exponentially more data. Mitigations: dimensionality reduction (PCA), feature selection, using L1 distance, or moving to ensemble methods.

3. Compare brute-force, KDTree, and BallTree in sklearn's kNN implementation.

Brute force O(nd) — fine for small datasets. KDTree partitions along axes, O(log n) average but degrades in high dimensions. BallTree uses hyperspheres and handles high dimensions better. sklearn auto-selects based on data.

4. How would you handle categorical features in kNN?

Convert to numeric via one-hot encoding, then use Hamming distance. Alternatively, use distance metrics designed for mixed data like Gower distance.

5. What is the effect of using distance-weighted vs. uniform voting?

Uniform voting gives all k neighbors equal say. Distance weighting assigns higher influence to closer neighbors, reducing the impact of distant noisy points and often improving accuracy, especially when data density varies.

### Advanced

1. Derive the influence function for kNN regression. How does changing k affect the effective degrees of freedom?

The effective degrees of freedom for kNN is approximately n/k. Each prediction averages k neighbors, reducing variance by factor 1/k. The influence matrix has entries w_ij = 1/k if j is among i's k nearest, 0 otherwise. Trace of this matrix gives DoF.

2. Prove that the kNN classifier is universally consistent under certain conditions.

As n→∞, if k → ∞ and k/n → 0, the kNN classifier's risk approaches the Bayes optimal risk. Proof uses the fact that the ball around x_q shrinks with n, and within that ball the conditional probability of each class converges.

3. Describe the Condensed Nearest Neighbor (CNN) algorithm and its purpose.

CNN selects a minimal subset of training data (prototypes) that still correctly classifies the rest. It iteratively adds misclassified points to the prototype set. This reduces storage and inference cost while preserving decision boundaries — a form of instance selection.

## Solutions

Solutions for Easy problems:

1. Iris k tuning:

```python
from sklearn.datasets import load_iris
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import cross_val_score
from sklearn.model_selection import train_test_split
import numpy as np

X, y = load_iris(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

best_k, best_score = 0, 0
for k in range(1, 31):
    knn = KNeighborsClassifier(n_neighbors=k)
    scores = cross_val_score(knn, X_train, y_train, cv=5)
    mean_score = scores.mean()
    if mean_score > best_score:
        best_score = mean_score
        best_k = k

print(f"Best k = {best_k} with CV accuracy = {best_score:.3f}")
# Output: Best k = 8 with CV accuracy = 0.971

final_model = KNeighborsClassifier(n_neighbors=best_k)
final_model.fit(X_train, y_train)
test_acc = final_model.score(X_test, y_test)
print(f"Test accuracy: {test_acc:.3f}")
# Output: Test accuracy: 0.978
```

2. kNN from scratch:

```python
import numpy as np
from collections import Counter

def knn_predict(X_train, y_train, X_test, k=5, weighted=False):
    predictions = []
    for x in X_test:
        dists = np.sqrt(np.sum((X_train - x) ** 2, axis=1))
        nearest = np.argsort(dists)[:k]
        nearest_dists = dists[nearest]
        nearest_labels = y_train[nearest]

        if not weighted:
            pred = Counter(nearest_labels).most_common(1)[0][0]
        else:
            weights = 1.0 / (nearest_dists + 1e-8)
            weight_sum = {}
            for label, w in zip(nearest_labels, weights):
                weight_sum[label] = weight_sum.get(label, 0) + w
            pred = max(weight_sum, key=weight_sum.get)
        predictions.append(pred)
    return np.array(predictions)

# Test
from sklearn.datasets import make_classification
X, y = make_classification(n_samples=200, n_features=4, random_state=42)
split = 150
preds = knn_predict(X[:split], y[:split], X[split:], k=7)
accuracy = np.mean(preds == y[split:])
print(f"From-scratch kNN accuracy: {accuracy:.3f}")
# Output: From-scratch kNN accuracy: 0.920
```

## Summary

k-Nearest Neighbors is a simple, instance-based learning algorithm that makes predictions by finding the k closest training examples to a query point. It is a lazy learner that stores all training data and computes distances at inference time. The key hyperparameter k controls the bias-variance tradeoff: small k leads to low bias, high variance; large k increases bias but reduces variance. Feature scaling is essential for kNN to work properly, and the algorithm suffers from the curse of dimensionality in high-dimensional spaces. Despite these limitations, kNN remains a powerful baseline and is widely used for its simplicity, interpretability, and strong performance on well-structured low-dimensional data.

## Key Takeaways

- kNN is a non-parametric, lazy learner — no training phase, all computation at inference
- k controls model complexity through the bias-variance tradeoff
- Feature scaling (standardization or normalization) is mandatory for kNN
- Distance weighting improves robustness to noise and outliers
- The curse of dimensionality severely degrades kNN in high dimensions
- Cross-validate to select k; typical values range from 3 to 20
- sklearn provides efficient implementations via KDTree and BallTree
- kNN works for both classification and regression
