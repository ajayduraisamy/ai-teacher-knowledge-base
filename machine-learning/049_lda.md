## Definition

Linear Discriminant Analysis (LDA) is a supervised linear dimensionality reduction technique that finds a projection maximizing class separability. It computes directions (linear discriminants) that maximize the ratio of between-class scatter to within-class scatter.

Given data with C classes, LDA seeks a projection matrix W that maximizes:

J(W) = |W^T S_B W| / |W^T S_W W|

where S_B is the between-class scatter matrix and S_W is the within-class scatter matrix. The optimal W consists of the eigenvectors of S_W^{-1} S_B corresponding to the largest eigenvalues. LDA projects to at most C-1 dimensions.

## Intuition

Imagine plotting points from two classes. PCA would find the direction of maximum total variance, which might not separate the classes well. LDA explicitly finds the direction that best separates the classes — maximizing the distance between class means while minimizing the spread within each class.

For a classification problem with C classes, LDA finds a subspace (at most C-1 dimensions) where the classes are maximally separated. It then uses this subspace for both visualization and classification (via Bayes decision rule with Gaussian class-conditional distributions sharing a common covariance).

## Why This Concept Matters

LDA is the go-to method for supervised dimensionality reduction. It is widely used in face recognition (Fisherfaces), medical diagnosis, text classification, and any application where reducing dimensionality while preserving class separation is important. LDA also serves as a classifier (linear decision boundary) in its own right. Understanding LDA provides insight into the relationship between dimensionality reduction, classification, and the geometry of class-conditional distributions.

## Mathematical Explanation

### Scatter Matrices

**Within-class scatter**:
S_W = sum_{k=1}^C sum_{x in class k} (x - mu_k)(x - mu_k)^T

**Between-class scatter**:
S_B = sum_{k=1}^C n_k (mu_k - mu)(mu_k - mu)^T

where mu_k is the mean of class k, mu is the overall mean, and n_k is the number of points in class k.

**Total scatter**: S_T = S_W + S_B

### Optimization

LDA maximizes the Rayleigh quotient:

J(w) = (w^T S_B w) / (w^T S_W w)

Taking derivative w.r.t. w and setting to zero:

S_B w = lambda S_W w => S_W^{-1} S_B w = lambda w

So w is a generalized eigenvector of (S_B, S_W). The top C-1 eigenvectors form the projection matrix.

### LDA as Classifier

LDA assumes each class has a Gaussian distribution with shared covariance matrix Sigma = S_W / (n - C). The decision boundary is:

delta_k(x) = x^T Sigma^{-1} mu_k - 0.5 mu_k^T Sigma^{-1} mu_k + log(pi_k)

This is linear in x (hence "Linear" Discriminant Analysis).

### Relationship to PCA

- PCA is unsupervised (ignores class labels), maximizes variance
- LDA is supervised (uses labels), maximizes class separation
- PCA is better for reconstruction; LDA is better for classification
- LDA can project to at most C-1 dimensions; PCA can project to d dimensions

## Code Examples

### Example 1: LDA for Dimensionality Reduction and Classification

```python
import numpy as np
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.datasets import load_iris, load_wine
from sklearn.model_selection import cross_val_score, train_test_split
from sklearn.preprocessing import StandardScaler

iris = load_iris()
X, y = iris.data, iris.target
X_scaled = StandardScaler().fit_transform(X)

lda = LinearDiscriminantAnalysis(n_components=2)
X_lda = lda.fit_transform(X_scaled, y)

print(f"Explained variance ratio: {lda.explained_variance_ratio_}")
# Output: Explained variance ratio: [0.9912126  0.0087874]

print(f"Scree plot: PC1 captures {lda.explained_variance_ratio_[0]:.1%} of between-class variance")
# Output: Scree plot: PC1 captures 99.1% of between-class variance

print(f"LDA components shape: {lda.scalings_.shape}")  # [n_features, n_components]
# Output: LDA components shape: (4, 2)

# LDA for classification
lda_clf = LinearDiscriminantAnalysis()
scores = cross_val_score(lda_clf, X_scaled, y, cv=5)
print(f"LDA classifier CV accuracy: {scores.mean():.3f}")
# Output: LDA classifier CV accuracy: 0.980
```

### Example 2: LDA vs PCA on Classification

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.decomposition import PCA

X, y = load_wine(return_X_y=True)
X_scaled = StandardScaler().fit_transform(X)

# LDA projection (max 2 components for 3 classes)
lda = LinearDiscriminantAnalysis(n_components=2)
X_lda = lda.fit_transform(X_scaled, y)

# PCA projection (2 components)
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

# Classify in reduced space
rf_lda = RandomForestClassifier(random_state=42).fit(X_lda, y)
rf_pca = RandomForestClassifier(random_state=42).fit(X_pca, y)

# Evaluate on test set
X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size=0.3, random_state=42)

lda_transformer = LinearDiscriminantAnalysis(n_components=2).fit(X_train, y_train)
pca_transformer = PCA(n_components=2).fit(X_train)

X_lda_train = lda_transformer.transform(X_train)
X_lda_test = lda_transformer.transform(X_test)
X_pca_train = pca_transformer.transform(X_train)
X_pca_test = pca_transformer.transform(X_test)

rf_lda = RandomForestClassifier(random_state=42).fit(X_lda_train, y_train)
rf_pca = RandomForestClassifier(random_state=42).fit(X_pca_train, y_train)

print(f"Test accuracy with LDA features: {rf_lda.score(X_lda_test, y_test):.3f}")
print(f"Test accuracy with PCA features: {rf_pca.score(X_pca_test, y_test):.3f}")
# Output:
# Test accuracy with LDA features: 0.981
# Test accuracy with PCA features: 0.926
```

### Example 3: LDA Decision Boundary

```python
import numpy as np
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.datasets import make_classification
import matplotlib.pyplot as plt

X, y = make_classification(n_samples=200, n_features=2, n_classes=3,
                           n_redundant=0, n_clusters_per_class=1, random_state=42)

lda = LinearDiscriminantAnalysis()
lda.fit(X, y)

x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.05),
                     np.arange(y_min, y_max, 0.05))
Z = lda.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)

plt.figure(figsize=(8, 6))
plt.contourf(xx, yy, Z, alpha=0.3, cmap='coolwarm')
plt.scatter(X[:, 0], X[:, 1], c=y, edgecolors='k', cmap='coolwarm')
plt.title('LDA Decision Boundaries (linear)')
plt.xlabel('Feature 1')
plt.ylabel('Feature 2')
plt.tight_layout()
plt.savefig('lda_boundary.png', dpi=150)
print("LDA boundary plot saved.")
# Output: LDA boundary plot saved.

# Linear decision regions are visible — LDA assumes shared covariance
```

### Example 5: Face Recognition with LDA

```python
from sklearn.datasets import fetch_olivetti_faces
from sklearn.model_selection import train_test_split
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.metrics import accuracy_score

faces = fetch_olivetti_faces()
X, y = faces.data, faces.target

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

# LDA for classification (40 classes, so max 39 components)
lda = LinearDiscriminantAnalysis()
lda.fit(X_train, y_train)
y_pred = lda.predict(X_test)
acc = accuracy_score(y_test, y_pred)
print(f"LDA face recognition accuracy: {acc:.3f}")
# Output: LDA face recognition accuracy: 0.975

# With PCA preprocessing (n_components=100 first)
from sklearn.decomposition import PCA
pca = PCA(n_components=80)
X_train_pca = pca.fit_transform(X_train)
X_test_pca = pca.transform(X_test)
lda_pca = LinearDiscriminantAnalysis()
lda_pca.fit(X_train_pca, y_train)
y_pred_pca = lda_pca.predict(X_test_pca)
print(f"PCA+LDA face recognition accuracy: {accuracy_score(y_test, y_pred_pca):.3f}")
# Output: PCA+LDA face recognition accuracy: 0.958
```

## Common Mistakes

1. **Using LDA when classes are not Gaussian with shared covariance.** LDA assumes each class follows a Gaussian with the same covariance. If these assumptions are violated, QDA or non-linear methods may work better.

2. **Applying LDA to very high-dimensional data (d >> n) without PCA first.** S_W becomes singular when d > n. Use PCA to reduce to n-1 dimensions first, then apply LDA.

3. **Using LDA with only 2 classes.** LDA projects to 1 dimension (C-1 = 1), which is a single direction. This may lose information.

4. **Expecting LDA to work well with non-linear separability.** LDA is linear. For non-linear boundaries, use QDA, kernel methods, or neural networks.

5. **Forgetting to use the class prior in LDA prediction.** LDA incorporates class priors pi_k. With imbalanced data, the prior affects the decision boundary.

6. **Using LDA as dimensionality reduction without cross-validation.** The number of components matters for downstream tasks. Validate via cross-validation.

7. **Assuming LDA components are interpretable like PCA components.** LDA components maximize class separation, not variance. They are linear combinations optimized for discrimination.

## Interview Questions

### Beginner

1. What is LDA?
Linear Discriminant Analysis is a supervised dimensionality reduction technique that finds projections maximizing class separation by maximizing between-class scatter relative to within-class scatter.

2. How does LDA differ from PCA?
PCA is unsupervised (maximizes variance), LDA is supervised (maximizes class separation). PCA is better for reconstruction; LDA is better for classification.

3. To how many dimensions can LDA reduce?
At most C-1 dimensions, where C is the number of classes. For binary classification, it projects to 1 dimension.

4. What assumptions does LDA make?
Each class follows a Gaussian distribution with shared covariance matrix (homoscedasticity). The decision boundary is linear.

5. Can LDA be used for classification?
Yes. LDA computes linear decision boundaries using Bayes rule with Gaussian class-conditional densities and shared covariance.

### Intermediate

1. Derive the LDA optimization problem.
Maximize J(w) = (w^T S_B w) / (w^T S_W w). This is the ratio of between-class to within-class variance after projection. Solution: generalized eigenvectors of (S_B, S_W).

2. What happens when S_W is singular in LDA?
When d > n, S_W is singular and non-invertible. Solutions: use PCA to reduce dimensions first, use pseudo-inverse, or use regularized LDA.

3. Compare LDA, QDA, and GaussianNB.
LDA: shared full covariance, linear boundary. QDA: per-class full covariance, quadratic boundary. GaussianNB: per-class diagonal covariance, quadratic boundary. LDA has fewest parameters; QDA most flexible.

4. How does LDA handle multi-class problems?
It projects to C-1 dimensions (or fewer) using the top eigenvectors of S_W^{-1} S_B. For classification, it uses multi-class linear discriminant functions.

5. What is Fisher's discriminant?
For binary classification, Fisher's discriminant is the 1D projection w = S_W^{-1} (mu_1 - mu_2). This maximizes the ratio of between-class to within-class variance.

### Advanced

1. Prove that the LDA eigenvectors are the generalized eigenvectors of S_B and S_W.
The Lagrangian for the constrained optimization: L = w^T S_B w - lambda (w^T S_W w - 1). dL/dw = 2S_B w - 2lambda S_W w = 0 => S_B w = lambda S_W w => S_W^{-1} S_B w = lambda w. So w is a generalized eigenvector.

2. Derive the relationship between LDA and linear regression with one-hot encoding.
For binary classification, LDA is equivalent to linear regression of the class indicator variable scaled by the class covariance. For multi-class, the connection is through optimal scoring.

3. Explain the bias-variance tradeoff between LDA and QDA.
LDA assumes shared covariance (high bias, low variance) — good with limited data per class. QDA estimates per-class covariance (low bias, high variance) — better with sufficient data. The choice depends on sample size relative to dimensionality.

## Summary

Linear Discriminant Analysis is a supervised dimensionality reduction and classification method that finds a linear projection maximizing class separability. It maximizes the ratio of between-class to within-class scatter, projecting to at most C-1 dimensions. LDA assumes Gaussian class distributions with shared covariance, producing linear decision boundaries. It is widely used for supervised dimensionality reduction, visualization, and as a classifier.

## Key Takeaways

- LDA is supervised (uses class labels), unlike PCA which is unsupervised
- Maximizes between-class scatter / within-class scatter ratio
- Projects to at most C-1 dimensions
- Assumes Gaussian classes with shared covariance
- Produces linear decision boundaries
- Requires S_W to be invertible — use PCA pre-processing if d > n
- Works well for face recognition (Fisherfaces)
- QDA relaxes the shared covariance assumption
