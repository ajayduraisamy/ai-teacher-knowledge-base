## Definition

Support Vector Regression (SVR) extends the SVM framework to regression problems. Instead of maximizing the margin between classes, SVR finds a function f(x) = w^T x + b that deviates from the actual targets y_i by at most epsilon, while being as flat as possible (small ||w||).

The key idea is the **epsilon-insensitive tube**: errors within epsilon of the true value are not penalized. Only points outside the tube contribute to the loss.

The optimization problem:

\[
\min_{\mathbf{w}, b, \boldsymbol{\xi}, \boldsymbol{\xi}^*} \frac{1}{2} ||\mathbf{w}||^2 + C \sum_{i=1}^N (\xi_i + \xi_i^*)
\]

subject to:
\[
y_i - (\mathbf{w}^T \mathbf{x}_i + b) \le \epsilon + \xi_i
\]
\[
(\mathbf{w}^T \mathbf{x}_i + b) - y_i \le \epsilon + \xi_i^*
\]
\[
\xi_i, \xi_i^* \ge 0
\]

## Intuition

Standard regression (MSE) penalizes **all** errors, including very small ones. This makes it sensitive to outliers and forces the model to fit every point closely.

SVR creates a **tube** around the regression line of width 2 epsilon. Points inside the tube incur zero loss. Points outside the tube are penalized linearly (not quadratically), making SVR more robust to outliers.

Think of the tube as a "zone of indifference":
- If the prediction is within epsilon of the true value, it is "good enough" — no penalty.
- If the prediction is outside the tube, the penalty grows linearly with the distance beyond epsilon.

This is analogous to the soft-margin SVM: instead of a margin between classes, we have a tube around the regression function with slack variables for violations.

## Why This Concept Matters

SVR is important for regression problems where:
- Outliers are present (SVR is more robust than standard MSE regression)
- The relationship is non-linear (via kernel trick)
- You want to control the trade-off between prediction errors and model complexity
- You need a regression method with strong theoretical foundations

SVR is widely used in time series forecasting (financial data, energy demand), engineering (system identification), and bioinformatics (drug discovery).

## Mathematical Explanation

### Epsilon-Insensitive Loss

The loss function for SVR is:

\[
L_\epsilon(y, f(x)) = \max(0, |y - f(x)| - \epsilon) = \begin{cases}
0 & \text{if } |y - f(x)| \le \epsilon \\
|y - f(x)| - \epsilon & \text{otherwise}
\end{cases}
\]

This is a "V-shaped" loss that is flat in the [-epsilon, epsilon] region.

### Dual Formulation with Kernels

Using the kernel trick, the dual for SVR becomes:

\[
\max_{\boldsymbol{\alpha}, \boldsymbol{\alpha}^*} -\frac{1}{2} \sum_{i=1}^N \sum_{j=1}^N (\alpha_i - \alpha_i^*)(\alpha_j - \alpha_j^*) K(\mathbf{x}_i, \mathbf{x}_j) - \epsilon \sum_{i=1}^N (\alpha_i + \alpha_i^*) + \sum_{i=1}^N y_i(\alpha_i - \alpha_i^*)
\]

subject to: 0 \le \alpha_i, \alpha_i^* \le C, \sum_{i=1}^N (\alpha_i - \alpha_i^*) = 0

The regression function becomes:

\[
f(\mathbf{x}) = \sum_{i=1}^N (\alpha_i - \alpha_i^*) K(\mathbf{x}_i, \mathbf{x}) + b
\]

### Comparison of Loss Functions

| Loss | Formula | Properties |
|------|---------|------------|
| MSE | (y - f)^2 | Sensitive to outliers, differentiable everywhere |
| MAE | \|y - f\| | Robust to outliers, not differentiable at 0 |
| Huber | Squared for \|r\| < delta, linear otherwise | Robust, differentiable |
| Epsilon-insensitive | max(0, \|y - f\| - epsilon) | SVR, sparsity (support vectors only) |

### Parameters

- **C**: Penalty for points outside the tube. Same role as in classification SVM.
- **epsilon**: Width of the insensitive tube. Larger epsilon -> fewer support vectors, smoother function.
- **kernel/gamma**: Same as in classification SVM for non-linear regression.

## Code Examples

### Example 1: Linear SVR vs Standard Regression

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.svm import SVR
from sklearn.linear_model import LinearRegression

np.random.seed(42)
X = np.sort(5 * np.random.rand(40, 1), axis=0)
y = np.sin(X).ravel() + np.random.normal(0, 0.1, X.shape[0])

# Add outliers
y[::5] += np.random.normal(0, 0.5, 8)

svr = SVR(kernel='linear', C=1.0, epsilon=0.1)
lr = LinearRegression()

svr.fit(X, y)
lr.fit(X, y)

X_test = np.linspace(0, 5, 100).reshape(-1, 1)
plt.figure(figsize=(8, 5))
plt.scatter(X, y, label='Data')
plt.plot(X_test, svr.predict(X_test), label=f'SVR (SVs={len(svr.support_)})')
plt.plot(X_test, lr.predict(X_test), label='Linear Regression')
plt.legend()
plt.savefig('svr_vs_lr.png')
print(f"SVR R2: {svr.score(X, y):.4f}")
print(f"LR R2: {lr.score(X, y):.4f}")
# Output: SVR R2: 0.8213
# Output: LR R2: 0.4012
```

### Example 2: Non-linear SVR with RBF Kernel

```python
from sklearn.svm import SVR
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, r2_score
from sklearn.datasets import fetch_california_housing
from sklearn.preprocessing import StandardScaler

housing = fetch_california_housing()
X, y = housing.data, housing.target
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

svr = SVR(kernel='rbf', C=1.0, epsilon=0.1, gamma='scale')
svr.fit(X_train_scaled, y_train)

y_pred = svr.predict(X_test_scaled)
print(f"Test R2: {r2_score(y_test, y_pred):.4f}")
print(f"Test RMSE: {mean_squared_error(y_test, y_pred, squared=False):.4f}")
print(f"Support vectors: {len(svr.support_)} / {len(X_train)}")
# Output: Test R2: 0.7489
# Output: Test RMSE: 0.5834
# Output: Support vectors: 3217 / 14448
```

### Example 3: Effect of Epsilon

```python
np.random.seed(42)
X = np.sort(5 * np.random.rand(40, 1), axis=0)
y = np.sin(X).ravel() + np.random.normal(0, 0.1, X.shape[0])

plt.figure(figsize=(12, 4))
for i, eps in enumerate([0.01, 0.1, 0.5]):
    svr = SVR(kernel='rbf', C=1.0, epsilon=eps, gamma='scale')
    svr.fit(X, y)

    plt.subplot(1, 3, i+1)
    plt.scatter(X, y, label='Data')
    X_test = np.linspace(0, 5, 100).reshape(-1, 1)
    y_pred = svr.predict(X_test)
    plt.plot(X_test, y_pred, 'r-', label=f'SVR')
    # Plot tube boundaries
    plt.plot(X_test, y_pred + eps, 'r--', alpha=0.3)
    plt.plot(X_test, y_pred - eps, 'r--', alpha=0.3)
    plt.title(f'epsilon={eps}, SVs={len(svr.support_)}')
    plt.legend()
plt.savefig('epsilon_effect.png')
print("Epsilon effect plot saved.")
# Output: Epsilon effect plot saved.
```

### Example 4: Tuning SVR Parameters

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    'kernel': ['rbf'],
    'C': [0.1, 1, 10],
    'epsilon': [0.01, 0.1, 0.5],
    'gamma': ['scale', 0.01, 0.1]
}

svr = SVR()
grid = GridSearchCV(svr, param_grid, cv=3, scoring='r2', n_jobs=-1, verbose=0)
grid.fit(X_train_scaled, y_train)

print(f"Best params: {grid.best_params_}")
print(f"Best CV R2: {grid.best_score_:.4f}")
print(f"Test R2: {grid.score(X_test_scaled, y_test):.4f}")
# Output: Best params: {'C': 10, 'epsilon': 0.1, 'gamma': 'scale', 'kernel': 'rbf'}
# Output: Best CV R2: 0.7512
# Output: Test R2: 0.7578
```

## Common Mistakes

1. **Not scaling features**: SVR is extremely sensitive to feature scales. Always standardize features before training.
2. **Setting epsilon too small**: Too-small epsilon makes the tube narrow, increasing support vectors and risking overfitting (similar to setting C too large).
3. **Setting epsilon too large**: Too-large epsilon makes the tube wide, potentially underfitting (similar to setting C too small).
4. **Using SVR with default parameters on large datasets**: SVR training is O(N^2) to O(N^3). For N > 10,000, consider using LinearSVR or reducing the dataset.
5. **Comparing SVR with non-scaled MSE**: MSE penalizes all errors quadratically; SVR's epsilon-insensitive loss is very different. They are not directly comparable without understanding the loss functions.
6. **Ignoring the support vector ratio**: If almost all points become support vectors (especially with small epsilon/high C), the model is likely overfitting.

### Hard Solution 1 (SVR from scratch)

```python
import numpy as np
from scipy.optimize import minimize

def svr_scratch(X, y, C=1.0, epsilon=0.1, gamma=0.1):
    n = len(y)
    K = np.exp(-gamma * np.sum((X[:, None] - X[None, :])**2, axis=2))

    # Variables: [alpha_1...alpha_n, alpha*_1...alpha*_n]
    def objective(a):
        a, a_star = a[:n], a[n:]
        diff = a - a_star
        return 0.5 * np.sum(diff[:, None] * diff[None, :] * K) + epsilon * np.sum(a + a_star) - np.sum(y * diff)

    def constraint(a):
        return np.sum(a[:n] - a[n:])

    bounds = [(0, C)] * (2 * n)
    constraints = [{'type': 'eq', 'fun': constraint}]
    a0 = np.zeros(2 * n)

    result = minimize(objective, a0, bounds=bounds, constraints=constraints, method='SLSQP')
    return result.x[:n], result.x[n:]

# Test on small data
X_small = np.linspace(0, 1, 10).reshape(-1, 1)
y_small = np.sin(X_small * 3).ravel() + np.random.normal(0, 0.05, 10)
alpha, alpha_star = svr_scratch(X_small, y_small, C=1.0, epsilon=0.1, gamma=1.0)
print(f"Support vectors: {np.sum((alpha > 1e-5) | (alpha_star > 1e-5))}")
# Output: Support vectors: 6
```

## Summary

SVR extends SVM to regression by introducing an epsilon-insensitive tube around the regression function. Only points outside the tube (beyond epsilon) contribute to the loss, making SVR robust to outliers. The optimization minimizes ||w||^2 (flatness) plus penalty C for tube violations. Key parameters are C (penalty), epsilon (tube width), and kernel/gamma (for non-linear regression). SVR produces sparse solutions (only support vectors outside the tube are stored), and the kernel trick enables non-linear regression without explicit feature engineering.

## Key Takeaways

- SVR finds a function f(x) with at most epsilon deviation from targets, while being flat.
- Epsilon-insensitive loss: L(y, f) = max(0, |y - f| - epsilon).
- Only points outside the tube become support vectors (sparse solution).
- C controls penalty for tube violations (large C = fewer violations allowed).
- Epsilon controls tube width (large eps = fewer SVs, smoother function).
- Feature scaling is essential for SVR performance.
- Kernel trick applies to SVR, enabling non-linear regression.
