## Definition

A Gaussian Mixture Model (GMM) is a probabilistic generative model that assumes data points are generated from a mixture of K Gaussian distributions, each with its own mean mu_k, covariance Sigma_k, and mixing coefficient pi_k (where sum pi_k = 1).

P(x) = sum_{k=1}^K pi_k * N(x | mu_k, Sigma_k)

GMM is used for soft clustering (each point has probability of belonging to each cluster), density estimation, and as a building block for more complex generative models.

## Intuition

Imagine data generated by several distinct processes, each producing normally distributed outputs with different centers and spreads. If you see all the data mixed together but don't know which process produced which point, GMM simultaneously estimates the parameters of each Gaussian and the probability that each point belongs to each Gaussian.

Unlike k-Means which assigns each point to exactly one cluster, GMM provides a probability distribution over cluster assignments. A point halfway between two clusters might be 60% cluster A and 40% cluster B rather than forced into one.

## Why This Concept Matters

GMMs are the most widely used mixture model and form the foundation for probabilistic clustering, density estimation, and generative modeling. They capture elliptical clusters of different sizes and orientations (unlike k-Means which assumes spherical clusters). GMMs are used in image segmentation, speaker recognition, anomaly detection, and as components in more complex models like variational autoencoders and hidden Markov models.

## Mathematical Explanation

### Model Definition

A GMM with K components:

P(x) = sum_{k=1}^K pi_k * N(x | mu_k, Sigma_k)

where:
- pi_k >= 0, sum pi_k = 1 (mixing coefficients)
- N(x | mu, Sigma) = 1/sqrt((2pi)^d |Sigma|) * exp(-0.5 (x - mu)^T Sigma^{-1} (x - mu))

### EM Algorithm for GMM

Goal: maximize log-likelihood sum_i log P(x_i | pi, mu, Sigma)

**E-step**: Compute responsibilities (posterior probability that point i belongs to component k):

gamma_{ik} = P(z_i = k | x_i) = pi_k * N(x_i | mu_k, Sigma_k) / sum_{j=1}^K pi_j * N(x_i | mu_j, Sigma_j)

**M-step**: Update parameters:

pi_k = (1/N) sum_i gamma_{ik}

mu_k = (sum_i gamma_{ik} * x_i) / (sum_i gamma_{ik})

Sigma_k = (sum_i gamma_{ik} * (x_i - mu_k)(x_i - mu_k)^T) / (sum_i gamma_{ik})

### Model Selection

AIC = 2 * num_params - 2 * log L
BIC = num_params * log(N) - 2 * log L

Lower values indicate better models, with BIC penalizing complexity more heavily.

### Covariance Types

- 'full': each component has its own full covariance matrix (most flexible)
- 'tied': all components share the same covariance matrix
- 'diag': each component has a diagonal covariance matrix
- 'spherical': each component has a single variance (spherical, like k-Means)

## Code Examples

### Example 1: Basic GMM Clustering

```python
import numpy as np
from sklearn.mixture import GaussianMixture
from sklearn.datasets import make_blobs
from sklearn.metrics import adjusted_rand_score

X, y = make_blobs(n_samples=500, centers=3, cluster_std=0.8, random_state=42)

gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=42)
gmm.fit(X)
labels = gmm.predict(X)
probs = gmm.predict_proba(X)

print(f"ARI vs true labels: {adjusted_rand_score(y, labels):.3f}")
# Output: ARI vs true labels: 1.000
print(f"Log-likelihood: {gmm.score(X):.3f}")
# Output: Log-likelihood: -1.597

print("Learned means:")
print(gmm.means_)
# Output:
# Learned means:
# [[-5.0694699   6.19998232]
#  [ 0.09791142 -4.95636225]
#  [ 6.99090765  0.8632783 ]]

print("Learned covariance matrices (first component):")
print(gmm.covariances_[0])
# Output:
# Learned covariance matrices (first component):
# [[0.5739121  0.02148056]
#  [0.02148056 0.63801063]]

# Soft clustering example for first 5 points
print("Soft assignments (first 5 points):")
for i in range(5):
    print(f"  Point {i}: cluster={labels[i]}, probs={np.round(probs[i], 3)}")
# Output:
# Soft assignments (first 5 points):
#   Point 0: cluster=1, probs=[0. 1. 0.]
#   Point 1: cluster=0, probs=[1. 0. 0.]
#   Point 2: cluster=0, probs=[1. 0. 0.]
#   Point 3: cluster=2, probs=[0. 0. 1.]
#   Point 4: cluster=0, probs=[1. 0. 0.]
```

### Example 2: GMM vs k-Means on Elliptical Clusters

```python
from sklearn.cluster import KMeans
import numpy as np

# Create elongated clusters (non-spherical)
np.random.seed(42)
X_ellipse = np.vstack([
    np.random.multivariate_normal([0, 0], [[3, 0], [0, 0.3]], 150),
    np.random.multivariate_normal([5, 5], [[0.5, 0], [0, 3]], 150),
    np.random.multivariate_normal([-3, 5], [[2, 1.5], [1.5, 2]], 150)
])

gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=42)
gmm_labels = gmm.fit_predict(X_ellipse)

kmeans = KMeans(n_clusters=3, random_state=42)
kmeans_labels = kmeans.fit_predict(X_ellipse)

# Since no ground truth, compute silhouette scores
from sklearn.metrics import silhouette_score
print(f"GMM silhouette: {silhouette_score(X_ellipse, gmm_labels):.3f}")
print(f"k-Means silhouette: {silhouette_score(X_ellipse, kmeans_labels):.3f}")
# Output:
# GMM silhouette: 0.614
# k-Means silhouette: 0.402
```

### Example 3: Covariance Type Comparison

```python
from sklearn.mixture import GaussianMixture
from sklearn.datasets import load_digits
from sklearn.decomposition import PCA

digits = load_digits()
X_pca = PCA(n_components=10).fit_transform(digits.data)

for cov_type in ['spherical', 'diag', 'tied', 'full']:
    gmm = GaussianMixture(n_components=10, covariance_type=cov_type, random_state=42, n_init=3)
    gmm.fit(X_pca)
    bic = gmm.bic(X_pca)
    aic = gmm.aic(X_pca)
    log_lik = gmm.score(X_pca)
    print(f"{cov_type:10s}: Log-lik={log_lik:.2f}, AIC={aic:.0f}, BIC={bic:.0f}")
# Output:
# spherical : Log-lik=-7.72, AIC=40988, BIC=41122
# diag      : Log-lik=-6.91, AIC=37298, BIC=37666
# tied      : Log-lik=-7.56, AIC=40716, BIC=41015
# full      : Log-lik=-5.23, AIC=31700, BIC=34238
```

### Example 4: Model Selection with BIC

```python
bics = []
aics = []
K_range = range(1, 11)

X, _ = make_blobs(n_samples=500, centers=4, cluster_std=0.6, random_state=42)

for k in K_range:
    gmm = GaussianMixture(n_components=k, covariance_type='full', random_state=42, n_init=3)
    gmm.fit(X)
    bics.append(gmm.bic(X))
    aics.append(gmm.aic(X))
    print(f"k={k}: AIC={aics[-1]:.0f}, BIC={bics[-1]:.0f}")
# Output:
# k=1: AIC=5766, BIC=5787
# k=2: AIC=4013, BIC=4051
# k=3: AIC=3073, BIC=3129
# k=4: AIC=2499, BIC=2572
# k=5: AIC=2502, BIC=2593
# k=6: AIC=2504, BIC=2612
# k=7: AIC=2509, BIC=2634
# k=8: AIC=2515, BIC=2658
# k=9: AIC=2522, BIC=2682
# k=10: AIC=2528, BIC=2705

best_k = K_range[np.argmin(bics)]
print(f"Optimal k by BIC: {best_k}")
# Output: Optimal k by BIC: 4
```

### Example 5: Density Estimation with GMM

```python
import numpy as np
from sklearn.mixture import GaussianMixture
from scipy.stats import norm

np.random.seed(42)
X_1d = np.concatenate([np.random.normal(-3, 1, 300),
                       np.random.normal(3, 1.5, 200)]).reshape(-1, 1)

gmm = GaussianMixture(n_components=2, random_state=42)
gmm.fit(X_1d)

print("Learned parameters:")
for k in range(2):
    print(f"  Component {k}: pi={gmm.weights_[k]:.3f}, "
          f"mu={gmm.means_[k,0]:.3f}, std={np.sqrt(gmm.covariances_[k,0,0]):.3f}")
# Output:
# Learned parameters:
#   Component 0: pi=0.592, mu=-2.929, std=0.998
#   Component 1: pi=0.408, mu=2.992, std=1.434

x_test = np.array([[-5], [0], [5]])
log_densities = gmm.score_samples(x_test)
print(f"Log-densities: {np.round(log_densities, 3)}")
# Output: Log-densities: [-6.274 -2.418 -2.41 ]
```

## Common Mistakes

1. **Assuming GMM finds globally optimal parameters.** EM converges to local optima. Use multiple random initializations (n_init).

2. **Choosing too many components.** Overfitting with low-density components. Use BIC/AIC for model selection.

3. **Ignoring covariance type.** Full covariance fits d(d+1)/2 parameters per component. With limited data, use diag or spherical.

4. **Using GMM for non-Gaussian clusters.** GMM assumes elliptical clusters. For crescent or spiral shapes, GMM fails.

5. **Not standardizing features.** Mixed-scale features cause numerical issues in covariance estimation.

6. **Treating GMM as hard clustering.** The strength of GMM is soft assignments. Use predict_proba, not just predict.

7. **Covariance singularity.** With few points per component, covariances can become singular. Increase regularization (reg_covar parameter).

## Interview Questions

### Beginner

1. What is a Gaussian Mixture Model?
A probabilistic model that assumes data is generated from a mixture of K Gaussian distributions. Each component has its own mean, covariance, and mixing weight.

2. How does GMM differ from k-Means?
GMM provides soft cluster assignments (probabilities), captures elliptical clusters of varying sizes, and estimates cluster orientation. k-Means gives hard assignments and assumes spherical clusters.

3. What is the EM algorithm?
Expectation-Maximization alternates between computing cluster responsibilities (E-step) and updating parameters (M-step) to maximize log-likelihood.

4. What are the parameters of a GMM?
Mixing coefficients pi_k, means mu_k, and covariance matrices Sigma_k for each of K components.

5. How do you choose K in GMM?
Use BIC or AIC (lower is better), silhouette score, or domain knowledge. BIC penalizes complexity more heavily.

### Intermediate

1. Derive the E-step for GMM.
P(z_i = k | x_i) = pi_k * N(x_i | mu_k, Sigma_k) / sum_j pi_j * N(x_i | mu_j, Sigma_j). This is the posterior probability of component k given data point x_i.

2. Explain why EM converges but may not find global optimum.
EM monotonically increases the log-likelihood (proven via Jensen's inequality) but converges to a local optimum. Multiple restarts with different initializations are needed.

3. Compare covariance types in sklearn's GMM.
'spherical': single variance per component (1 param). 'diag': diagonal covariance (d params). 'tied': shared full covariance. 'full': independent full covariance per component (most flexible, d(d+1)/2 params).

4. How does GMM handle overlapping clusters?
Through soft assignments. A point in the overlap region has non-zero probability of belonging to multiple clusters, weighted by the relative densities.

5. What is the relationship between GMM and k-Means?
k-Means is a limiting case of GMM where covariances are spherical and equal, and assignments become hard (responsibilities approach 0 or 1).

### Advanced

1. Prove that the EM algorithm monotonically increases log-likelihood for GMM.
The log-likelihood lower bound is: log P(X|theta) >= ELBO(q, theta) = sum_i sum_k q(z_i=k) log [P(x_i,z_i=k|theta)/q(z_i=k)] + KL(q || P(z|X,theta)). The E-step sets q = P(z|X,theta_old), making the bound tight. The M-step maximizes the bound w.r.t. theta, ensuring log P(X|theta_new) >= log P(X|theta_old).

2. How can GMMs be extended to handle high-dimensional data with limited samples?
Use factor analyzers (mixture of factor analyzers), which model each covariance as Sigma_k = Lambda_k Lambda_k^T + Psi_k (low-rank plus diagonal). This reduces parameters from O(d^2) to O(dq) per component.

3. Derive the M-step updates for GMM parameters.
For pi_k: maximize Lagrangian L = sum_i sum_k gamma_ik log pi_k + lambda (sum_k pi_k - 1). Taking derivative gives pi_k = (1/N) sum_i gamma_ik. For mu_k: set derivative of Q = sum_i gamma_ik (x_i - mu_k)^T Sigma_k^{-1} (x_i - mu_k) to zero, giving mu_k = sum gamma_ik x_i / sum gamma_ik. Similar for Sigma_k.

## Summary

Gaussian Mixture Models are probabilistic generative models for soft clustering and density estimation. They extend k-Means by allowing elliptical clusters, soft assignments, and probabilistic density modeling. Parameters are estimated via the Expectation-Maximization algorithm, which iteratively computes responsibilities (E-step) and updates parameters (M-step). Model selection uses BIC or AIC to choose the number of components. GMMs are powerful but computationally intensive for high dimensions.

## Key Takeaways

- GMM is a probabilistic soft clustering model with Gaussian components
- EM algorithm iteratively learns parameters and assignments
- Covariance type controls flexibility: spherical < diag < tied < full
- Use BIC/AIC for model selection (number of components)
- GMM handles elliptical clusters of varying sizes (unlike k-Means)
- EM converges monotonically to a local optimum — use multiple restarts
- GMM provides both cluster assignments and density estimates
- Regularization (reg_covar) prevents covariance singularity
