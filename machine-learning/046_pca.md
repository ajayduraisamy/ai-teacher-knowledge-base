## Definition

Principal Component Analysis (PCA) is a linear dimensionality reduction technique that projects data onto a lower-dimensional subspace while preserving as much variance as possible. It finds the directions (principal components) of maximum variance in the data and projects the data onto these directions.

Given centered data X (n x d), PCA computes the eigendecomposition of the covariance matrix C = X^T X / n and selects the top k eigenvectors (principal components) that capture the most variance.

## Intuition

Imagine a cloud of points in 3D space shaped like a flattened cigar. PCA finds the longest axis of the cigar (first principal component), then the second longest axis perpendicular to the first, and so on. By projecting onto just the first two axes, you capture almost all the variance while reducing from 3D to 2D.

PCA can be thought of as finding a new coordinate system where the first axis points in the direction of highest variance, the second in the next highest direction orthogonal to the first, and so on. This reveals the natural structure of the data.

## Why This Concept Matters

PCA is the most fundamental dimensionality reduction technique. It is used for visualization (projecting high-dimensional data to 2D/3D), noise reduction (discarding low-variance components), feature extraction, preprocessing for other algorithms, and data compression. Understanding PCA provides the foundation for understanding more advanced methods and the role of eigenvalues in data analysis.

## Mathematical Explanation

### Two Equivalent Formulations

**Maximum variance formulation**: Find unit vector w such that Var(Xw) is maximized.

**Minimum reconstruction error formulation**: Find projection P_k onto k-dimensional subspace that minimizes sum_i ||x_i - P_k(x_i)||^2.

Both formulations lead to the same solution.

### Steps

1. Center the data: X_centered = X - mean(X)
2. Compute covariance matrix: C = (1/n) X_centered^T X_centered
3. Eigendecomposition: C v_j = lambda_j v_j, where lambda_1 >= lambda_2 >= ... >= lambda_d
4. Select top k eigenvectors to form projection matrix W_k (d x k)
5. Project: Z = X_centered W_k

### Explained Variance Ratio

The fraction of total variance captured by the j-th component:

explained_variance_ratio_j = lambda_j / sum_{i=1}^d lambda_i

### Using SVD

Instead of eigendecomposition of C, PCA can use the SVD of X_centered:

X_centered = U Sigma V^T

The principal components are the columns of V, and the explained variance is proportional to sigma_j^2.

### Whitening

PCA whitening transforms components to have unit variance:

Z_whitened = Z * diag(1/sqrt(lambda_j))

This removes correlations between components.

## Code Examples

### Example 1: Basic PCA on Iris Data

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.decomposition import PCA
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler

iris = load_iris()
X, y = iris.data, iris.target
target_names = iris.target_names

X_scaled = StandardScaler().fit_transform(X)

pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

print(f"Explained variance ratio: {pca.explained_variance_ratio_}")
# Output: Explained variance ratio: [0.72770452 0.23030523]
print(f"Total variance explained: {pca.explained_variance_ratio_.sum():.4f}")
# Output: Total variance explained: 0.9580
print(f"Components:\n{pca.components_}")
# Output:
# Components:
# [[ 0.52237162 -0.26335492  0.58125401  0.56561105]
#  [ 0.37231836  0.92555649  0.02109478  0.06541577]]

# 2D projection
plt.figure(figsize=(8, 6))
colors = ['navy', 'turquoise', 'darkorange']
for color, i, target_name in zip(colors, [0, 1, 2], target_names):
    plt.scatter(X_pca[y == i, 0], X_pca[y == i, 1], color=color, alpha=0.8, lw=2, label=target_name)
plt.legend()
plt.title("PCA of Iris Dataset (2D projection)")
plt.xlabel(f"PC1 ({pca.explained_variance_ratio_[0]:.1%} variance)")
plt.ylabel(f"PC2 ({pca.explained_variance_ratio_[1]:.1%} variance)")
plt.tight_layout()
plt.savefig('pca_iris.png', dpi=150)
print("Iris PCA plot saved.")
# Output: Iris PCA plot saved.
```

### Example 2: Explained Variance vs Number of Components

```python
import numpy as np
from sklearn.datasets import load_digits
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

digits = load_digits()
X = digits.data
pca = PCA()
pca.fit(X)

cumulative_variance = np.cumsum(pca.explained_variance_ratio_)
print("Components needed for variance thresholds:")
for threshold in [0.5, 0.7, 0.8, 0.9, 0.95, 0.99]:
    n_components = np.argmax(cumulative_variance >= threshold) + 1
    print(f"  {threshold:.0%} variance: {n_components} components")
# Output:
#   50% variance: 6 components
#   70% variance: 13 components
#   80% variance: 20 components
#   90% variance: 33 components
#   95% variance: 45 components
#   99% variance: 64 components

print(f"\nTotal features: {X.shape[1]}")
print(f"With 33 components, we retain 90% of variance (3x reduction)")
# Output: Total features: 64
# Output: With 33 components, we retain 90% of variance (3x reduction)
```

### Example 3: PCA for Noise Reduction

```python
import numpy as np
from sklearn.datasets import load_digits
from sklearn.decomposition import PCA
from sklearn.metrics import mean_squared_error

digits = load_digits()
X = digits.data

noise = np.random.normal(0, 10, X.shape)
X_noisy = X + noise

pca = PCA(n_components=32)
X_reduced = pca.fit_transform(X_noisy)
X_denoised = pca.inverse_transform(X_reduced)

mse_noisy = mean_squared_error(X, X_noisy)
mse_denoised = mean_squared_error(X, X_denoised)

print(f"MSE with noise: {mse_noisy:.2f}")
print(f"MSE after PCA denoising: {mse_denoised:.2f}")
print(f"Improvement: {(1 - mse_denoised/mse_noisy) * 100:.1f}%")
# Output:
# MSE with noise: 99.56
# MSE after PCA denoising: 30.95
# Improvement: 68.9%
```

### Example 4: PCA from Scratch vs sklearn

```python
import numpy as np
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# Generate data
np.random.seed(42)
X = np.random.randn(100, 5) @ np.diag([4, 3, 2, 1, 0.5])
X_centered = X - X.mean(axis=0)

# From scratch via eigendecomposition
cov = (X_centered.T @ X_centered) / (X_centered.shape[0] - 1)
eigvals, eigvecs = np.linalg.eigh(cov)
idx = np.argsort(eigvals)[::-1]
eigvals_sorted = eigvals[idx]
eigvecs_sorted = eigvecs[:, idx]
X_pca_scratch = X_centered @ eigvecs_sorted[:, :2]

# sklearn
pca = PCA(n_components=2)
X_pca_sk = pca.fit_transform(X)

# Compare (may have sign flips)
correlation = np.corrcoef(np.abs(X_pca_scratch[:, 0]), np.abs(X_pca_sk[:, 0]))[0, 1]
print(f"Correlation between scratch and sklearn PC1: {correlation:.6f}")
# Output: Correlation between scratch and sklearn PC1: 1.000000

print(f"Eigenvalues (from scratch): {np.round(eigvals_sorted, 3)}")
print(f"Explained variance (sklearn): {np.round(pca.explained_variance_, 3)}")
# Output:
# Eigenvalues (from scratch): [17.711  9.684  4.133  1.068  0.249]
# Explained variance (sklearn): [17.711  9.684  4.133  1.068  0.249]
```

### Example 5: PCA as Preprocessing for Classification

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

digits = load_digits()
X, y = digits.data, digits.target

# Without PCA
rf = RandomForestClassifier(random_state=42)
scores_no_pca = cross_val_score(rf, X, y, cv=5).mean()

# With PCA (95% variance)
pca = PCA(n_components=0.95)
X_pca = pca.fit_transform(X)
scores_with_pca = cross_val_score(rf, X_pca, y, cv=5).mean()

# With PCA (reduction to 30 components)
pca30 = PCA(n_components=30)
X_pca30 = pca30.fit_transform(X)
scores_pca30 = cross_val_score(rf, X_pca30, y, cv=5).mean()

print(f"No PCA ({X.shape[1]} feats): {scores_no_pca:.4f}")
print(f"PCA 95% ({X_pca.shape[1]} feats): {scores_with_pca:.4f}")
print(f"PCA 30D ({X_pca30.shape[1]} feats): {scores_pca30:.4f}")
# Output:
# No PCA (64 feats): 0.9355
# PCA 95% (45 feats): 0.9399
# PCA 30D (30 feats): 0.9410
```

## Common Mistakes

1. **Not centering the data.** PCA on uncentered data finds the first principal component pointing toward the mean, not the direction of maximum variance.

2. **Applying PCA without scaling.** Features with larger variance dominate the first principal components. Always standardize if features are on different scales.

3. **Assuming PCA components are interpretable.** Principal components are linear combinations of all original features. They rarely have a clear physical meaning.

4. **Using PCA for supervised feature selection.** PCA is unsupervised — it doesn't know about the target variable. Components with high variance may be useless for prediction.

5. **Retaining too few components.** If you discard components with meaningful signal, you lose predictive power. Use cross-validation to tune the number of components.

6. **Forgetting that PCA assumes linearity.** PCA only captures linear correlations. For non-linear data, use kernel PCA, t-SNE, or UMAP.

7. **Misinterpreting explained variance.** High explained variance does not guarantee good performance for downstream tasks.

## Interview Questions

### Beginner

1. What is PCA?
A linear dimensionality reduction technique that projects data onto the directions of maximum variance, creating new features (principal components) that are uncorrelated.

2. What are principal components?
Orthogonal directions of maximum variance. PC1 captures the most variance, PC2 the most remaining variance orthogonal to PC1, etc.

3. How do you choose the number of components?
Use the explained variance ratio — pick enough components to capture 80-95% of total variance. Alternatively, use the elbow in the scree plot.

4. Why must data be centered for PCA?
Centering ensures the first PC points in the direction of maximum variance. Without centering, the first PC points toward the mean, which is not the direction of maximum spread.

5. What is the explained variance ratio?
The fraction of total dataset variance captured by each component. It equals the component's eigenvalue divided by the sum of all eigenvalues.

### Intermediate

1. Explain the relationship between PCA and SVD.
PCA can be computed via SVD of the centered data matrix: X = U Sigma V^T. The principal components are the columns of V, and the eigenvalues are sigma_j^2 / (n-1). SVD is numerically more stable than eigendecomposition of the covariance matrix.

2. How does PCA relate to linear regression? Both minimize squared error, but PCA minimizes orthogonal distances (perpendicular to the subspace), while regression minimizes vertical distances (perpendicular to the axis).

3. What is the difference between PCA and Factor Analysis?
PCA explains total variance and creates components that are linear combinations of observed variables. Factor Analysis explains common variance only and models observed variables as linear combinations of latent factors plus unique variance.

4. How does PCA help with multicollinearity?
PCA creates orthogonal (uncorrelated) components, eliminating multicollinearity. Ridge regression is another approach that handles correlated features without reducing dimensionality.

5. Can PCA be used for outlier detection?
Yes. Outliers have large reconstruction error when projected back from the low-dimensional subspace, or have extreme projections onto later components (which capture low-variance directions).

### Advanced

1. Prove that the eigenvectors of the covariance matrix maximize variance.
For symmetric C, maximize w^T C w subject to w^T w = 1. Lagrangian: L = w^T C w - lambda(w^T w - 1). dL/dw = 2Cw - 2lambda w = 0 => Cw = lambda w. So w is an eigenvector of C, and the variance is w^T C w = lambda. The first PC is the eigenvector with largest eigenvalue.

2. Derive the relationship between PCA and autoencoders with linear activations.
A linear autoencoder with k hidden units minimizes reconstruction error: ||X - X W W^T||^2. This is equivalent to PCA when W has orthonormal columns. The optimal W spans the same subspace as the top k eigenvectors of the covariance matrix.

3. How does kernel PCA extend PCA to non-linear data?
Kernel PCA maps data to a higher-dimensional feature space via phi(x) and performs PCA in that space using the kernel trick. The kernel matrix K_ij = k(x_i, x_j) is eigendecomposed instead of the covariance matrix. This captures non-linear structure.

## Summary

PCA is a linear dimensionality reduction technique that projects data onto directions of maximum variance via eigendecomposition of the covariance matrix or SVD. It is widely used for visualization, noise reduction, and preprocessing. Key considerations include feature scaling, the number of components (based on explained variance), and the assumption of linearity. PCA is unsupervised — it does not use label information.

## Key Takeaways

- PCA finds orthogonal directions of maximum variance
- Compute via eigendecomposition of covariance or SVD of centered data
- Scale features before PCA unless they share the same units
- Explained variance ratio guides component selection
- PCA is linear — use kernel PCA for non-linear structure
- PCA is unsupervised — components may not be predictive
- PCA can denoise by removing low-variance components
- Scree plot visualizes the importance of each component
- Whitening normalizes component variances
