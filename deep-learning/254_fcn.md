## Definition

Fully Convolutional Networks (FCNs), introduced by Long, Shelhamer, and Darrell in 2015, adapt classification CNNs (AlexNet, VGG, GoogLeNet) for dense pixel-wise prediction by replacing fully connected layers with convolutional layers and adding upsampling stages. FCNs accept arbitrary input sizes and produce correspondingly-sized output maps. The key innovation is the use of skip connections that combine coarse semantic information from deep layers with fine appearance information from shallow layers, producing detailed and accurate segmentation maps.

## Intuition

A classification CNN compresses an image into a single vector, losing spatial information. FCN reinterprets this: instead of predicting a class for the whole image, it converts the final feature map back to pixel space through learned upsampling. The challenge is that deep features have rich semantic information but poor spatial resolution, while shallow features have good resolution but limited semantics. Skip connections fuse features from multiple depths: the deep layer provides "what" the object is, and the shallow layer provides "where" its boundaries are.

## Why This Concept Matters

FCN was the first deep learning architecture to achieve end-to-end, pixel-to-pixel semantic segmentation, establishing the convolutional paradigm for dense prediction. It demonstrated that classification networks could be repurposed for segmentation with relatively minimal modification. The concepts of fully convolutional design, learned upsampling with transpose convolutions, and skip connections are foundational to virtually all subsequent segmentation architectures (U-Net, DeepLab, PSPNet).

## Mathematical Explanation

FCN converts fully connected layers to convolutions. For VGG-16:
- fc6 (4096 units) → Conv7x7 with 4096 filters
- fc7 (4096 units) → Conv1x1 with 4096 filters
- fc8 (1000 units) → Conv1x1 with num_classes filters

Up-sampling path:
- FCN-32s: Upsample stride-32 feature map 32x in one step
- FCN-16s: Combine pool4 features (stride 16) with upsampled pool5 (2x), then upsample 16x
- FCN-8s: Combine pool3 features (stride 8) with upsampled pool4, then upsample 8x

Each upsampling uses transpose convolution (deconvolution) with fixed or learned filters.

Loss: Pixel-wise cross-entropy, summed over all spatial positions:
L = Σ_i Σ_j Σ_c y_ij^c log(p_ij^c)

## Code Examples

### Example 1: Converting a Classification Net to FCN

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class FCN32s(nn.Module):
    def __init__(self, num_classes=21):
        super().__init__()
        # Encoder (VGG-16 inspired, simplified)
        self.conv1 = nn.Sequential(
            nn.Conv2d(3, 64, 3, padding=1), nn.ReLU(),
            nn.Conv2d(64, 64, 3, padding=1), nn.ReLU(),
        )
        self.pool1 = nn.MaxPool2d(2)  # 1/2

        self.conv2 = nn.Sequential(
            nn.Conv2d(64, 128, 3, padding=1), nn.ReLU(),
            nn.Conv2d(128, 128, 3, padding=1), nn.ReLU(),
        )
        self.pool2 = nn.MaxPool2d(2)  # 1/4

        self.conv3 = nn.Sequential(
            nn.Conv2d(128, 256, 3, padding=1), nn.ReLU(),
            nn.Conv2d(256, 256, 3, padding=1), nn.ReLU(),
            nn.Conv2d(256, 256, 3, padding=1), nn.ReLU(),
        )
        self.pool3 = nn.MaxPool2d(2)  # 1/8

        self.conv4 = nn.Sequential(
            nn.Conv2d(256, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
        )
        self.pool4 = nn.MaxPool2d(2)  # 1/16

        self.conv5 = nn.Sequential(
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
        )
        self.pool5 = nn.MaxPool2d(2)  # 1/32

        # Fc6 and fc7 as convolutions
        self.fc6 = nn.Conv2d(512, 4096, 7)  # 7x7 conv (instead of fc)
        self.fc7 = nn.Conv2d(4096, 4096, 1)  # 1x1 conv
        self.score = nn.Conv2d(4096, num_classes, 1)

        self.dropout = nn.Dropout(0.5)

    def forward(self, x):
        h = self.conv1(x)
        h = self.pool1(h)
        h = self.conv2(h)
        h = self.pool2(h)
        h = self.conv3(h)
        h = self.pool3(h)
        h = self.conv4(h)
        h = self.pool4(h)
        h = self.conv5(h)
        h = self.pool5(h)

        h = self.dropout(F.relu(self.fc6(h)))
        h = self.dropout(F.relu(self.fc7(h)))
        h = self.score(h)

        # Upsample 32x
        h = F.interpolate(h, scale_factor=32, mode='bilinear', align_corners=False)
        return h

model = FCN32s(num_classes=21)
dummy = torch.randn(1, 3, 256, 256)
out = model(dummy)
print(f"FCN-32s output: {out.shape}")
# Output: FCN-32s output: torch.Size([1, 21, 256, 256])
```

### Example 2: FCN-8s with Skip Connections

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class FCN8s(nn.Module):
    def __init__(self, num_classes=21):
        super().__init__()
        # Encoder (same as FCN-32s)
        self.conv1 = nn.Sequential(
            nn.Conv2d(3, 64, 3, padding=1), nn.ReLU(),
            nn.Conv2d(64, 64, 3, padding=1), nn.ReLU(),
        )
        self.pool1 = nn.MaxPool2d(2)

        self.conv2 = nn.Sequential(
            nn.Conv2d(64, 128, 3, padding=1), nn.ReLU(),
            nn.Conv2d(128, 128, 3, padding=1), nn.ReLU(),
        )
        self.pool2 = nn.MaxPool2d(2)

        self.conv3 = nn.Sequential(
            nn.Conv2d(128, 256, 3, padding=1), nn.ReLU(),
            nn.Conv2d(256, 256, 3, padding=1), nn.ReLU(),
            nn.Conv2d(256, 256, 3, padding=1), nn.ReLU(),
        )
        self.pool3 = nn.MaxPool2d(2)  # Save skip connection here (stride 8)

        self.conv4 = nn.Sequential(
            nn.Conv2d(256, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
        )
        self.pool4 = nn.MaxPool2d(2)  # Save skip connection here (stride 16)

        self.conv5 = nn.Sequential(
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
            nn.Conv2d(512, 512, 3, padding=1), nn.ReLU(),
        )
        self.pool5 = nn.MaxPool2d(2)

        # Classifier convolutions
        self.fc6 = nn.Conv2d(512, 4096, 7)
        self.fc7 = nn.Conv2d(4096, 4096, 1)
        self.score = nn.Conv2d(4096, num_classes, 1)

        # Skip connection scoring
        self.score_pool4 = nn.Conv2d(512, num_classes, 1)
        self.score_pool3 = nn.Conv2d(256, num_classes, 1)

    def forward(self, x):
        h = self.conv1(x)
        h = self.pool1(h)
        h = self.conv2(h)
        h = self.pool2(h)
        h = self.conv3(h)
        pool3_out = h  # Save for skip
        h = self.pool3(h)

        h = self.conv4(h)
        pool4_out = h  # Save for skip
        h = self.pool4(h)

        h = self.conv5(h)
        h = self.pool5(h)

        h = F.relu(self.fc6(h))
        h = F.relu(self.fc7(h))
        h = self.score(h)  # [N, C, H/32, W/32]

        # Fuse with pool4 (stride 16)
        h = F.interpolate(h, scale_factor=2, mode='bilinear', align_corners=False)
        pool4_score = self.score_pool4(pool4_out)
        h = h + pool4_score

        # Fuse with pool3 (stride 8)
        h = F.interpolate(h, scale_factor=2, mode='bilinear', align_corners=False)
        pool3_score = self.score_pool3(pool3_out)
        h = h + pool3_score

        # Upsample to input resolution
        h = F.interpolate(h, scale_factor=8, mode='bilinear', align_corners=False)
        return h

model = FCN8s(num_classes=21)
out = model(dummy)
print(f"FCN-8s output: {out.shape}")
# Output: FCN-8s output: torch.Size([1, 21, 256, 256])
```

### Example 3: Transpose Convolution Visualization

```python
import torch
import torch.nn as nn
import matplotlib.pyplot as plt
import numpy as np

def visualize_transpose_conv():
    # Simple 4x4 input
    x = torch.tensor([[[[1.0, 2.0], [3.0, 4.0]]]])
    # Transpose conv with 3x3 kernel, stride=2
    conv_t = nn.ConvTranspose2d(1, 1, 3, stride=2)
    # Set fixed weights for visualization
    conv_t.weight.data = torch.ones(1, 1, 3, 3)
    conv_t.bias.data.zero_()

    y = conv_t(x)
    print(f"Input: {x.shape}, Output: {y.shape}")
    print(f"Output values:\n{y[0, 0].detach().numpy()}")
    # Output:
    # Input: torch.Size([1, 1, 2, 2]), Output: torch.Size([1, 1, 5, 5])
    # Output values:
    # [[1. 3. 3. 2. 0.]
    #  [4. 10. 10. 6. 0.]
    #  [6. 14. 14. 8. 0.]
    #  [4. 8. 8. 4. 0.]
    #  [0. 0. 0. 0. 0.]]

visualize_transpose_conv()
```

## Common Mistakes

1. **Checkerboard artifacts from transpose convolution**: Using transpose convolution with kernel size not divisible by stride creates uneven overlap. Use bilinear interpolation upsampling followed by convolution, or ensure kernel size is a multiple of stride.

2. **Not handling the shift from fc to conv**: Converting fc6 (7×7 receptive field) requires careful handling of the receptive field size. The original FCN uses a 7×7 convolution that only works if the final feature map is at least 7×7.

3. **Single-scale upsampling loses detail**: FCN-32s produces blurry outputs. Skip connections (FCN-8s, FCN-16s) dramatically improve boundary quality by incorporating high-resolution features.

4. **Memory consumption from large feature maps**: Storing all intermediate feature maps for skip connections significantly increases memory usage. Trade off between memory and accuracy.

5. **Fixed input size from classifier conversion**: If the feature map becomes too small for the 7×7 conv (fc6) for small inputs, the model crashes. Use global pooling or adaptive convolutions.

def fcn_8s_forward(model, x):
    # Assume model has encoder layers
    pool3, pool4, pool5 = model.encoder(x)
    h = model.classifier(pool5)
    h = F.interpolate(h, scale_factor=2, mode='bilinear', align_corners=False)

## Summary

FCNs revolutionized semantic segmentation by adapting classification networks for dense prediction through fully convolutional design and transpose convolution upsampling. The introduction of skip connections to fuse semantic and spatial information (FCN-8s) dramatically improved boundary detail. FCN established the encoder-decoder paradigm that underpins modern segmentation architectures.

## Key Takeaways

- Fully convolutional design: no fully connected layers
- Classification → segmentation: convert fc layers to conv layers
- Upsampling via transpose convolution (deconvolution)
- Skip connections fuse coarse semantics with fine spatial detail
- FCN-32s < FCN-16s < FCN-8s in accuracy (more skip connections)
- Arbitrary input sizes supported
- Pixel-wise cross-entropy loss
- Foundation for U-Net, DeepLab, PSPNet, and all modern segmentation
