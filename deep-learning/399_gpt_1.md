## Definition

GPT-1 (Generative Pre-trained Transformer) is the first model in the GPT series, introduced by Radford et al. (2018). It is a 12-layer decoder-only Transformer with 117 million parameters, pre-trained on the BooksCorpus dataset using a standard language modeling objective (next token prediction). GPT-1 demonstrated that a single pre-trained language model could be fine-tuned to achieve state-of-the-art results on a wide range of NLP tasks, including text classification, natural language inference, question answering, and commonsense reasoning. The key innovation was showing that generative pre-training followed by discriminative fine-tuning — without task-specific architectural changes — could match or exceed carefully engineered task-specific models.

## Intuition

GPT-1's insight was deceptively simple: if you train a model to predict the next word on a massive amount of books, it learns an enormous amount about language — grammar, syntax, semantics, reasoning patterns, world knowledge — that can be leveraged for specific tasks. This is analogous to a person who reads extensively before specializing: the broad reading builds general competence that makes specialized training more effective.

The model architecture is minimal: just stacked decoder blocks with causal self-attention, feed-forward networks, and layer normalization. There are no task-specific components. When fine-tuning for classification, the input is simply formatted as "text [SEP] label" and the model predicts the label as the next token. This simplicity was revolutionary.

## Why This Concept Matters

GPT-1 established the pre-train-then-fine-tune paradigm for decoder-only models, parallel to BERT's encoder-only approach. It demonstrated that:

1. Generative pre-training produces useful representations for discriminative tasks.
2. A single model architecture can handle diverse tasks without modification.
3. Language modeling is a powerful self-supervised objective.
4. The decoder-only architecture is viable for both understanding and limited generation tasks.

GPT-1 directly led to GPT-2 (which showed scaling improves quality) and GPT-3 (which showed few-shot learning emerges at scale). It laid the foundation for the entire GPT ecosystem.

## Mathematical Explanation

### Pre-training Objective

Given a sequence of tokens U = (u_1, ..., u_n), the language modeling objective is:

L_1 = sum_i log P(u_i | u_{i-k}, ..., u_{i-1}; Theta)

Where k is the context window size (typically 512).

For the decoder output h_l at layer l:

h_0 = U W_e + W_p (token embeddings + position embeddings)
h_l = transformer_block(h_{l-1}) for l = 1 to L
P(u_i | u_{<i}) = softmax(h_L W_e^T)

### Fine-tuning Objective

For a supervised task with labeled examples (x_1, ..., x_m, y):

The input is transformed into a sequence using task-specific formatting:
- Classification: [Start] x [Delim] [Extract]
- Entailment: [Start] premise [Delim] hypothesis [Extract]
- QA: [Start] question [Delim] document [Extract]
- Similarity: [Start] text1 [Delim] text2 [Extract] and [Start] text2 [Delim] text1 [Extract]

The [Extract] token's final hidden state is fed through a linear classifier:

P(y | x_1, ..., x_m) = softmax(h_l_m^Extract W_y)

L_2 = sum_{(x,y)} log P(y | x)

### Combined Fine-tuning Loss

L_3 = L_2 + lambda * L_1

GPT-1 uses both the task-specific loss L_2 and the language modeling loss L_1 during fine-tuning as an auxiliary objective, which improves generalization.

## Code Examples

### Example 1: GPT-1 Style Input Formatting

```python
import torch
import torch.nn as nn

def format_for_classification(text, tokenizer, label=None):
    tokens = ["[CLS]"] + tokenizer.tokenize(text) + ["[SEP]"]
    if label is not None:
        tokens += [str(label)]
    return tokenizer.convert_tokens_to_ids(tokens)

def format_for_entailment(premise, hypothesis, tokenizer):
    tokens = ["[CLS]"] + tokenizer.tokenize(premise) + ["[DELIM]"] + \
             tokenizer.tokenize(hypothesis) + ["[EXTRACT]"]
    return tokenizer.convert_tokens_to_ids(tokens)

def format_for_qa(question, document, tokenizer):
    tokens = ["[CLS]"] + tokenizer.tokenize(question) + ["[DELIM]"] + \
             tokenizer.tokenize(document) + ["[EXTRACT]"]
    return tokenizer.convert_tokens_to_ids(tokens)

class MockTokenizer:
    @staticmethod
    def tokenize(text):
        return text.lower().split()

    @staticmethod
    def convert_tokens_to_ids(tokens):
        return [hash(t) % 1000 for t in tokens]

tokenizer = MockTokenizer()
cls_input = format_for_classification("I loved this movie!", tokenizer, label="pos")
print("Classification input:", cls_input)
# Output: Classification input: [210, 229, 561, 312, 783, 888, 104, 503]
print("Classification input length:", len(cls_input))
# Output: Classification input length: 8
```

### Example 2: GPT-1 Decoder Block

```python
class GPT1Block(nn.Module):
    def __init__(self, d_model=768, n_heads=12, d_ff=3072, dropout=0.1):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        self.mlp = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )

    def forward(self, x, mask=None):
        x = x + self.attn(self.ln1(x), self.ln1(x), self.ln1(x), attn_mask=mask)[0]
        x = x + self.mlp(self.ln2(x))
        return x

class GPT1Model(nn.Module):
    def __init__(self, vocab_size=40478, d_model=768, n_layers=12, n_heads=12, max_len=512):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.dropout = nn.Dropout(0.1)
        self.blocks = nn.ModuleList([
            GPT1Block(d_model, n_heads) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)
        self.lm_head.weight = self.token_emb.weight

    def forward(self, input_ids):
        B, T = input_ids.shape
        pos = torch.arange(T, device=input_ids.device).unsqueeze(0)
        x = self.dropout(self.token_emb(input_ids) + self.pos_emb(pos))
        mask = torch.triu(torch.full((T, T), float("-inf"), device=input_ids.device), diagonal=1)
        for block in self.blocks:
            x = block(x, mask)
        x = self.ln_f(x)
        return self.lm_head(x)

model = GPT1Model(vocab_size=1000, d_model=256, n_layers=6)
x = torch.randint(0, 1000, (2, 16))
logits = model(x)
print("GPT-1 logits shape:", logits.shape)
# Output: GPT-1 logits shape: torch.Size([2, 16, 1000])
print("Total parameters:", sum(p.numel() for p in model.parameters()))
# Output: Total parameters: 12822696
```

### Example 3: Fine-tuning GPT-1 for Classification

```python
class GPT1ForClassification(nn.Module):
    def __init__(self, gpt_model, n_classes=2, d_model=768):
        super().__init__()
        self.gpt = gpt_model
        self.classifier = nn.Linear(d_model, n_classes)

    def forward(self, input_ids):
        logits = self.gpt(input_ids)
        last_hidden = logits[:, -1, :]
        return self.classifier(last_hidden)
```

## Common Mistakes

1. Confusing GPT-1 with later GPT models: GPT-1 uses post-norm (LayerNorm after residual addition), following the original Transformer. GPT-2 and later switched to pre-norm for better training stability.

2. Not using the auxiliary LM loss during fine-tuning: GPT-1's fine-tuning includes the LM loss as an auxiliary objective. Omitting this can hurt generalization, especially on smaller fine-tuning datasets.

3. Misunderstanding the task formatting: GPT-1 transforms all tasks into text sequences with special tokens. The format varies by task, and using the wrong format degrades performance.

4. Using a too-small context window: GPT-1 uses a 512-token context. Processing sequences shorter than the model's maximum context may limit the model's ability to capture long-range dependencies.

5. Forgetting learning rate differences: GPT-1 uses different learning rates during pre-training (2.5e-4) and fine-tuning (6.25e-5). Using the pre-training learning rate during fine-tuning can cause catastrophic forgetting.

6. Assuming GPT-1 supports task-agnostic generation: GPT-1 is primarily designed for fine-tuning. Its generation capabilities are limited compared to GPT-2 and later models.

## Interview Questions

### Beginner

Q: What is the main contribution of GPT-1 in the history of NLP?

A: GPT-1 demonstrated that generative pre-training on a large unlabeled corpus (BooksCorpus), followed by discriminative fine-tuning on specific tasks, could achieve state-of-the-art performance across diverse NLP tasks without task-specific architectural changes. This established the pre-train-fine-tune paradigm for decoder-only models.

### Intermediate

Q: How does GPT-1 format inputs for different downstream tasks? Give two examples.

A: GPT-1 transforms all tasks into a sequence-to-sequence format. For text classification: [CLS] text [SEP] label. For natural language inference: [CLS] premise [DELIM] hypothesis [EXTRACT] entailment. For question answering: [CLS] question [DELIM] document [EXTRACT] answer. The special tokens indicate the start, separator, and extraction point, allowing a single model architecture to handle diverse tasks.

### Advanced

Q: GPT-1 uses an auxiliary language modeling loss during fine-tuning (L_1 + lambda * L_2). Why is this helpful, and how does it compare to BERT's fine-tuning approach?

A: The auxiliary LM loss serves as a regularizer during fine-tuning. By maintaining the LM objective alongside the task-specific loss, the model retains its general language understanding while adapting to the specific task. This prevents catastrophic forgetting and improves generalization, particularly when the fine-tuning dataset is small. BERT, in contrast, typically only uses the task-specific loss during fine-tuning. BERT can afford this because its bidirectional pre-training provides stronger representations per parameter. GPT-1's generative pre-training produces different representations, and the auxiliary loss helps maintain their quality during fine-tuning.

## Summary

GPT-1 is a 12-layer decoder-only Transformer pre-trained on BooksCorpus using language modeling and fine-tuned on diverse NLP tasks. It established the pre-train-fine-tune paradigm for decoder-only models, demonstrating that generative pre-training produces transferable representations for understanding tasks. The architecture uses post-norm, task-specific input formatting, and an auxiliary LM loss during fine-tuning.

## Key Takeaways

- GPT-1 is the first model in the GPT family with 117M parameters.
- Pre-trained on BooksCorpus with standard language modeling objective.
- Fine-tuned on diverse tasks using task-specific input formatting.
- Uses post-norm (LayerNorm after residual) unlike later GPT models.
- Auxiliary LM loss during fine-tuning improves generalization.
- Inputs formatted with [CLS], [DELIM], [EXTRACT] special tokens.
- Established the viability of decoder-only models for NLP.
- Direct predecessor to GPT-2 and GPT-3.
