## Definition

GPT-2 (Generative Pre-trained Transformer 2) is the second generation of OpenAI's GPT models, introduced by Radford et al. (2019). It is a scaled-up decoder-only Transformer available in four sizes: Small (117M), Medium (345M), Large (774M), and XL (1.5B parameters). GPT-2 introduced several architectural improvements over GPT-1: (1) pre-norm layer normalization (LayerNorm before each sublayer), (2) scaled initialization, (3) a larger byte-pair encoding vocabulary (50,257 tokens), and (4) increased context size (1024 tokens). The key finding was that language models could perform zero-shot task transfer — simply by conditioning the model on a task description (a "prompt"), it could perform the task without any supervised fine-tuning.

## Intuition

GPT-2's most surprising discovery was zero-shot transfer. The researchers noticed that by simply showing the model a pattern — like "Translate English to French: English text = French text" — the model could continue the pattern and perform translation without ever being explicitly trained for it. This was an emergent capability that appeared at scale.

Imagine showing someone a few examples of a puzzle and having them solve new instances without any explicit instruction. That is what GPT-2 demonstrated. The model learned so much about language structure during pre-training that task inference became embedded in its weights.

The scaling was also crucial. The 1.5B parameter GPT-2 XL was substantially larger than anything publicly available at the time, and its performance on zero-shot tasks was notably better than the smaller variants, demonstrating that scaling consistently improved task transfer.

## Why This Concept Matters

GPT-2 was a landmark model that changed how the field thought about language model capabilities:

1. Introduced zero-shot task transfer through prompt conditioning, laying the groundwork for in-context learning.
2. Demonstrated that scaling consistently improves language model capabilities.
3. Showed that a single model could perform diverse tasks without task-specific fine-tuning.
4. Released in stages due to concerns about misuse, sparking important discussions about responsible AI release.
5. The architectural improvements (pre-norm, scaled initialization) became standard for subsequent models.
6. Set new standards for text generation quality and coherence.

## Mathematical Explanation

### Architectural Changes from GPT-1

**Pre-Norm** (GPT-2):
x = x + Sublayer(LayerNorm(x))

**Initialization**:
- Residual projection weights scaled by 1/sqrt(2 * n_layer)
- Standard deviation = 0.02 for all other weights

**Vocabulary**:
Byte-level BPE with 50,257 tokens (vs GPT-1's 40,478 BPE tokens)

**Context**:
1024 tokens (vs GPT-1's 512)

### Model Sizes

| Variant | Layers | d_model | Heads | Parameters |
|---------|--------|---------|-------|------------|
| Small | 12 | 768 | 12 | 117M |
| Medium | 24 | 1024 | 16 | 345M |
| Large | 36 | 1280 | 20 | 774M |
| XL | 48 | 1600 | 25 | 1.5B |

### Zero-shot Transfer

GPT-2 demonstrates zero-shot transfer through the language modeling interface. Given a prompt that describes a task:

Q: "What is the capital of France?"
A: "Paris"

The model continues generating, conditioned on the prompt pattern. No gradients, no fine-tuning, no examples — just the conditioning signal from the prompt.

## Code Examples

### Example 1: GPT-2 Block (Pre-Norm)

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class GPT2Block(nn.Module):
    def __init__(self, d_model=768, n_heads=12, d_ff=3072, dropout=0.1):
        super().__init__()
        self.ln_1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.ln_2 = nn.LayerNorm(d_model)
        self.mlp = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )

    def forward(self, x, causal_mask=None):
        # Pre-norm: LayerNorm before attention and MLP, residual connection around them
        x = x + self.attn(self.ln_1(x), self.ln_1(x), self.ln_1(x), attn_mask=causal_mask)[0]
        x = x + self.mlp(self.ln_2(x))
        return x

class GPT2Model(nn.Module):
    def __init__(self, vocab_size=50257, d_model=768, n_layers=12, n_heads=12, max_len=1024):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.dropout = nn.Dropout(0.1)
        self.blocks = nn.ModuleList([
            GPT2Block(d_model, n_heads) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)
        self.lm_head.weight = self.token_emb.weight

    def forward(self, input_ids):
        B, T = input_ids.shape
        positions = torch.arange(T, device=input_ids.device).unsqueeze(0)
        x = self.dropout(self.token_emb(input_ids) + self.pos_emb(positions))
        mask = torch.triu(torch.full((T, T), float("-inf"), device=input_ids.device), diagonal=1)
        for block in self.blocks:
            x = block(x, mask)
        x = self.ln_f(x)
        return self.lm_head(x)

model = GPT2Model(vocab_size=1000, d_model=256, n_layers=6, max_len=256)
x = torch.randint(0, 1000, (2, 32))
logits = model(x)
print("GPT-2 style model logits:", logits.shape)
# Output: GPT-2 style model logits: torch.Size([2, 32, 1000])
print("Uses pre-norm: LN before each sublayer")
# Output: Uses pre-norm: LN before each sublayer
```

### Example 2: Scaled Initialization

```python
def scaled_init(module, n_layers, std=0.02):
    if isinstance(module, nn.Linear):
        if hasattr(module, "weight"):
            scaled_std = std / (2 * n_layers) ** 0.5
            nn.init.normal_(module.weight, mean=0.0, std=scaled_std)
        if hasattr(module, "bias") and module.bias is not None:
            nn.init.zeros_(module.bias)

class GPT2WithInit(nn.Module):
    def __init__(self, vocab_size=50257, d_model=768, n_layers=12, n_heads=12, max_len=1024):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.blocks = nn.ModuleList([GPT2Block(d_model, n_heads) for _ in range(n_layers)])
        self.ln_f = nn.LayerNorm(d_model)
        self.lm_head = nn.Linear(d_model, vocab_size)
        self.apply(lambda m: scaled_init(m, n_layers))
        nn.init.normal_(self.token_emb.weight, mean=0.0, std=0.02)
        nn.init.normal_(self.pos_emb.weight, mean=0.0, std=0.01)

model = GPT2WithInit(vocab_size=1000, d_model=256, n_layers=4)
print("Model initialized with GPT-2 scaling")
# Output: Model initialized with GPT-2 scaling
```

### Example 3: Zero-shot Prompting

```python
@torch.no_grad()
def zero_shot_task(model, prompt, tokenizer, max_new=20):
    input_ids = tokenizer.encode(prompt, return_tensors="pt")
    generated = input_ids.clone()

    for _ in range(max_new):
        logits = model(generated)
        next_logits = logits[:, -1, :]
        next_token = next_logits.argmax(dim=-1, keepdim=True)
        generated = torch.cat([generated, next_token], dim=-1)
        if next_token.item() == tokenizer.eos_token_id:
            break

    return tokenizer.decode(generated[0])

class MockTokenizer:
    def __init__(self):
        self.vocab = {f"tok_{i}": i for i in range(1000)}
        self.eos_token_id = 50256

    def encode(self, text, **kwargs):
        return torch.randint(0, 100, (1, 5))

    def decode(self, ids):
        return f"Generated text from {len(ids)} tokens"

tokenizer = MockTokenizer()
prompt = "Q: What is the capital of France?\nA:"
print("Zero-shot prompting demonstration")
# Output: Zero-shot prompting demonstration
print("GPT-2 can perform tasks without explicit training")
# Output: GPT-2 can perform tasks without explicit training
```

## Common Mistakes

1. Confusing GPT-2's pre-norm with GPT-1's post-norm: GPT-2 places LayerNorm before each sublayer (pre-norm), while GPT-1 places it after (post-norm). This is an important architectural difference that improves training stability in deeper models.

2. Forgetting scaled initialization: GPT-2 uses a specific initialization scheme where residual projection weights are scaled by 1/sqrt(2*n_layer). Omitting this can cause training instability, especially for larger variants.

3. Not recognizing that GPT-2 is NOT designed for fine-tuning: While fine-tuning is possible, GPT-2's strength is zero-shot and few-shot transfer through prompting. The original paper focused on this capability.

4. Underestimating the importance of the context size increase: GPT-2 increased context from 512 to 1024 tokens. This 2x increase significantly improved the model's ability to handle long-range dependencies and provide more in-context examples.

5. Confusing byte-level BPE with WordPiece: GPT-2 uses byte-level BPE which can represent any Unicode character without UNK tokens. WordPiece (used by BERT) requires all tokens to be in the vocabulary and uses the ## prefix for subword continuations.

6. Mishandling the vocabulary size: GPT-2's vocabulary is 50,257 tokens, which is about 65% larger than BERT's 30,522. This increases the embedding matrix and LM head size, contributing to the model's parameter count.

## Interview Questions

### Beginner

Q: What is the key architectural difference between GPT-1 and GPT-2?

A: The main architectural change is the placement of LayerNorm. GPT-1 uses post-norm (LayerNorm after the residual addition), while GPT-2 uses pre-norm (LayerNorm before each sublayer, with the residual connection around the sublayer). GPT-2 also has a larger vocabulary (50,257 vs 40,478 BPE tokens), a longer context window (1024 vs 512), and scaled initialization.

### Intermediate

Q: GPT-2 demonstrated zero-shot task transfer. Explain what this means and give an example.

A: Zero-shot task transfer means a model can perform a task it was never explicitly trained on by simply conditioning on a task description in the prompt. For example, by providing a prompt like "Translate English to French: I like cats = J'aime les chats. The house is big =" the model can generate the correct French translation without having been trained on translation pairs. The model has learned language structure so well during pre-training that it can infer task requirements from prompt patterns.

### Advanced

Q: GPT-2's paper title is "Language Models are Unsupervised Multitask Learners." Explain the theoretical basis for why a language model trained only on next-token prediction can perform multiple tasks zero-shot.

A: The key insight is that natural language itself contains the structure of many tasks. In a large corpus, you find sentences that implicitly demonstrate tasks: questions followed by answers, premises followed by conclusions, documents followed by summaries, etc. A sufficiently large language model trained on next-token prediction learns to recognize these task structures and deploy them when triggered by appropriate prompts. The language modeling objective naturally includes all tasks that can be expressed as text continuations. The model learns "which distribution to sample from" given the context. A question-answer context primes the model to sample from the answer distribution. This is possible because the training data contains abundant examples of task demonstrations embedded in natural text.

## Solutions

```python
# Easy solution
class GPT1PostNorm(nn.Module):
    def forward(self, x, attn, ffn):
        x = nn.LayerNorm(768)(x + attn(x))
        x = nn.LayerNorm(768)(x + ffn(x))
        return x

class GPT2PreNorm(nn.Module):
    def forward(self, x, attn, ffn):
        x = x + attn(nn.LayerNorm(768)(x))
        x = x + ffn(nn.LayerNorm(768)(x))
        return x

x = torch.randn(1, 8, 768)
attn = nn.MultiheadAttention(768, 12, batch_first=True)
ffn = nn.Sequential(nn.Linear(768, 3072), nn.GELU(), nn.Linear(3072, 768))

gpt1_block = GPT1PostNorm()
gpt2_block = GPT2PreNorm()

out1 = gpt1_block(x, attn, ffn)
out2 = gpt2_block(x, attn, ffn)
print("Outputs differ:", not torch.allclose(out1, out2))
# Output: Outputs differ: True
```

## Summary

GPT-2 scaled the decoder-only architecture from 117M to 1.5B parameters, introduced pre-norm layer normalization and scaled initialization, and demonstrated zero-shot task transfer through prompt conditioning. It showed that language models could perform diverse tasks without supervised fine-tuning, paving the way for GPT-3's in-context learning.

## Key Takeaways

- GPT-2 has four variants: Small (117M), Medium (345M), Large (774M), XL (1.5B).
- Key architectural changes from GPT-1: pre-norm, scaled initialization, larger vocabulary.
- Byte-level BPE tokenization with 50,257 tokens handles any text.
- Context window increased from 512 to 1024 tokens.
- Zero-shot task transfer through prompt conditioning.
- Scaling consistently improves language generation quality and task performance.
- The architectural innovations became standard for subsequent decoder-only models.
- GPT-2 laid the groundwork for GPT-3's in-context learning paradigm.
